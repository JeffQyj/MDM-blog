# MDM-blog 项目规则

## 1. 博客内容和主题风格一致性

### 1.1 内容风格

| 维度     | 规范                                                             |
| ------ | -------------------------------------------------------------- |
| **语言** | 统一使用简体中文（zh\_CN），变量/函数名保持英文                                    |
| **语气** | 第一视角、亲切、简洁、严谨、结构化、风趣幽默，撰写技术文章时，一定要保证不枯燥，代码示例一定要有注释，优先使用分点/表格呈现 |
| **排版** | 中文与英文/数字之间留空格，标点符号使用全角                                         |
| **标题** | 采用 Markdown 标准标题层级（`#` / `##` / `###`），避免跳级                    |

### 1.2 frontmatter 规范

所有博客文章必须包含以下 frontmatter 字段，且修改文章内容时无需修改frontmatter 字段内容，撰写文章前一定要**参考三篇以上同分类且内容具有相关性**的文章，并调用article-writing技能进行文章撰写：

```yaml
---
title: 文章标题
published: YYYY-MM-DD          # 发布日期
description: 简短描述（建议 50-150 字）
tags: [标签1, 标签2]            # 数组形式
category: 分类名称                # 单个分类
draft: false                     # true 表示草稿
lang: zh_CN                      # 语言代码
---
```

### 1.3 文章结构模板

项目支持 5 种 admonition 类型，分别适用于不同场景：

| 类型 | 使用场景 | 语义强度 |
|------|----------|----------|
| `tip` | 技巧提示、最佳实践 | 轻量提示 |
| `note` | 补充说明、一般注意事项 | 中性 |
| `important` | 重要内容、需要强调的信息 | 重要 |
| `caution` | 警告、需要小心处理的事项 | 警示 |
| `warning` | 严重警告、可能导致问题的事项 | 最高警示 |

```markdown
:::tip
文章开篇说明或核心要点提示
:::

## 章节标题

正文内容...

:::note
补充说明或注意事项
:::

## 下一章节

:::important
需要重点强调的内容
:::

:::caution
需要注意避免的坑或警示事项
:::

:::warning
严重警告或可能导致问题的内容
:::

---

*文章结尾，可添加延伸阅读*
```

### 1.4 主题配置一致性

- 站点元信息（标题、副标题、描述）统一在 `src/config.ts` 中管理
- 样式变量统一在 `src/styles/` 目录下管理
- 组件开发遵循现有组件的命名和结构模式

### 1.5 代码风格

- 代码风格遵循 `biome.json` 配置
- Astro/Svelte 文件使用 Tab 缩进
- JavaScript/TypeScript 使用双引号
- 运行 `pnpm check` 和 `pnpm format` 校验后再提交
