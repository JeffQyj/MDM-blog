## 3. 生涯补全系列规范

### 3.1 系列定位

「生涯补全」是一个回忆性质的系列文章，记录个人成长与技术学习历程中的关键节点。

### 3.2 时间认定规则

| 场景 | published | 正文时间 |
|------|-----------|----------|
| 一般博客文章 | 实际发布日期 | 故事发生时间在正文中体现 |
| 生涯补全系列 | 用户提供的故事发生时间 | 统一在正文章节中标注 |

:::note
生涯补全系列的时间以用户提供的故事发生时间为准，不受实际发布时间的限制。
:::

### 3.3 存放位置

生涯补全系列文章统一存放在 `src/content/posts/` 目录下。

```
src/content/posts/生涯补全：[具体事件/阶段标题].md
```

### 3.4 文件命名规范

生涯补全系列文章统一使用以下命名格式：

```
生涯补全：[具体事件/阶段标题].md
```

例如：

- `生涯补全：从Hello-World开始的编程之路.md`
- `生涯补全：xxx.md`

### 3.5 推送规范

推送时默认全量推送，直接执行 `git add .` 即可。推送前无需额外校验，仅确保文章内容符合规范。
