## 3. 生涯补全系列规范

### 3.1 系列定位

「生涯补全」是一个回忆性质的系列文章，记录个人成长与技术学习历程中的关键节点。

### 3.2 时间认定规则

| 场景     | published   | 正文时间         |
| ------ | ----------- | ------------ |
| 一般博客文章 | 实际发布日期      | 故事发生时间在正文中体现 |
| 生涯补全系列 | 用户提供的故事发生时间 | 统一在正文章节中标注   |

:::note
生涯补全系列的时间以用户提供的故事发生时间为准，不受实际发布时间的限制。
:::

### 3.3 存放位置

生涯补全系列文章统一存放在 `src/content/posts/` 目录下。

```
src/content/posts/生涯补全：[具体事件/阶段标题].md
```

### 3.4 文件命名规范

生涯补全系列文章统一使用以下命名格式：

```
生涯补全：[具体事件/阶段标题].md
```

例如：

- `生涯补全：从Hello-World开始的编程之路.md`
- `生涯补全：xxx.md`

### 3.5 推送规范

推送前需额外校验，推送语法严格遵守powershell语法，因为默认终端是powershell。推送时默认全量推送，直接执行 `git add .` 即可。

### 3.6 写作视角与个人设定

#### 3.6.1 写作视角

生涯补全系列文章采用「站在今天回顾过去」的叙事视角。

- 文章内容基于 2022 年 9 月开始的编程学习经历
- 怀念的是那个时间段的时光，而非更早的时期
- 整体基调是：对过去的回顾、感悟与珍重以及从现在往回看的感慨

#### 3.6.2 个人设定

| 维度       | 设定                               |
| -------- | -------------------------------- |
| **时间线**  | 2022 年 9 月开始大学生活，同期开始系统学习编程      |
| **学习背景** | 计算机专业，老师主导让我们通过网课（翁恺老师等）和互联网资源自学 |
| **经济条件** | 普通大学生，无经济困难，有电脑和网络资源             |

:::note
以上设定为生涯补全系列文章的默认背景设定，文章内容应与设定保持一致。
:::
