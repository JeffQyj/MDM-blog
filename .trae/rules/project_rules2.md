## 2. 博客文章生成流程

### 2.1 主题确认阶段

当用户提供博客主题后，先通过提问收集关键信息，而非直接生成。

| 问题类型 | 必问项 | 选问项 |
|---------|-------|-------|
| **时间线** | 事件发生的大致时间 | 具体日期是否已知 |
| **技术细节** | 使用的技术栈/工具 | 版本号、关键配置 |
| **个人感受** | 核心感悟或收获 | 有趣的细节、踩坑经历 |
| **期望篇幅** | 短/中/长 | — |

### 2.2 生成阶段

收集完信息后，按照以下顺序创作：

1. 确定文章结构和章节安排
2. 撰写符合项目风格的 frontmatter
3. 按照「[1.3 文章结构模板](#13-文章结构模板)」填充内容
4. 检查语言、格式是否符合规范

### 2.3 输出格式

生成后返回：

```
**文章概要：**
| 属性 | 内容 |
|------|------|
| 标题 | xxx |
| 发布日期 | YYYY-MM-DD |
| 分类 | xxx |
| 标签 | xxx |
| 篇幅 | 约 xxx 字 |

## 3. 文章内容规范

### 3.1 禁止互动类内容

本项目暂无留言、评论功能，文章中**禁止出现**以下互动引导：

| 类型 | 示例 |
|------|------|
| 征求留言 | 「欢迎留言分享」「有问题欢迎留言」 |
| 征求意见 | 「如果你有更好的方法，欢迎提出」 |
| 提问读者 | 「你有没有遇到过这种情况？」 |
| 征集分享 | 「欢迎分享你的经历」 |

文章结尾仅允许使用静态引导，如「有问题可提 Issue」或「更多内容可查看 xxx」。

### 3.2 日期语义规范

- `published` 字段表示**文章发布的时间**，而非故事发生时间
- 若文章内容为回忆性质，「故事发生时间」应在正文中体现，不体现在 frontmatter
- 示例：回忆 2022 年的故事，published 应为实际发布时间（如 2025年），正文开头写「2022年的某天...」
```
