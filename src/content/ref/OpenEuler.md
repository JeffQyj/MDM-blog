# 基于OpenEuler的轻量化安全态势感知系统项目内容分析报告

## 一、项目核心概述

本项目是面向国产操作系统环境的服务器安全监控与态势感知解决方案，针对传统系统监控工具界面不友好、可视化程度低、部署复杂、商业化产品成本高昂等行业痛点，以开源国产操作系统OpenEuler 24.03为底层基座，整合若依（RuoYi）前后端分离框架、Spring Boot后端技术、Vue3前端框架与ECharts可视化组件，设计并实现了一套集系统资源实时监控、运行状态可视化、安全威胁智能检测、自动化响应处置于一体的轻量化安全态势感知系统。

项目核心目标可归纳为四点：一是构建适配OpenEuler的轻量化监控平台，实现低资源占用、高运行效率；二是复用若依框架成熟的监控接口与后台管理能力，避免重复开发；三是通过ECharts实现监控数据与安全态势的可视化展示，降低使用门槛；四是打造从威胁检测到自动化响应的完整安全闭环，为服务器运维与安全管理提供技术支撑。

## 二、项目核心内容深度拆解

### （一）需求与可行性分析

项目从功能、非功能两个维度完成了全面的需求梳理，同时完成了技术、经济、操作三大维度的可行性论证，为系统设计与实现奠定了完整的需求边界。

1. **核心功能需求**
   项目划定了7大核心功能模块，可分为基础运维监控与安全态势感知两大板块：
   | 板块     | 核心功能   | 具体能力                                    |
   | ------ | ------ | --------------------------------------- |
   | 基础运维监控 | 系统信息监控 | 服务器基础信息、CPU/内存/磁盘/JVM实时状态监控与历史趋势追踪      |
   | 基础运维监控 | 缓存监控   | 缓存信息查询、手动清理、命中率统计分析                     |
   | 基础运维监控 | 定时任务监控 | 任务列表展示、执行日志查询、任务启停控制                    |
   | 基础运维监控 | 日志审计   | 操作日志、登录日志的多条件查询与追溯                      |
   | 基础运维监控 | 在线用户管理 | 在线用户列表展示、异常用户强制下线                       |
   | 基础运维监控 | 可视化展示  | 实时监控图表、统计图表、交互式数据查看                     |
   | 安全态势感知 | 核心安全能力 | 安全仪表盘、安全事件管理、安全规则管理、智能检测引擎、自动化响应、动态安全评分 |
2. **非功能需求指标**
   项目明确了可量化的非功能目标，核心指标如下：
   - 性能：页面平均加载时间＜2秒，数据刷新时间＜1.5秒，运行时CPU占用率＜10%，内存占用＜200MB；
   - 可靠性：支持7×24小时不间断运行，数据采集误差控制在可接受范围，具备完善的异常处理机制；
   - 安全性：完善的访问控制、密码加密存储、全流程操作日志审计能力；
   - 可用性：界面友好、操作便捷，兼容Chrome、Firefox、Edge等主流浏览器。
3. **可行性论证结论**
   - 技术可行性：所选技术栈均为成熟开源技术，文档完善、社区活跃，部署与开发流程可控；
   - 经济可行性：全栈采用开源免费技术，无软件授权成本，硬件要求低，开发与维护成本可控；
   - 操作可行性：界面符合用户操作习惯，功能聚焦、操作步骤简洁，学习门槛低，配套文档完善。

### （二）系统架构与设计

项目采用B/S架构与前后端分离设计模式，整体划分为三层架构，同时完成了功能模块与数据库的精细化设计，架构具备低耦合、高扩展性的特点。

1. **三层总体架构**
   | 架构层级  | 核心技术栈                                         | 核心职责                                            |
   | ----- | --------------------------------------------- | ----------------------------------------------- |
   | 前端展示层 | Vue3、Element UI、ECharts、Axios                 | 用户界面展示、数据可视化、用户交互处理、后端接口调用                      |
   | 后端服务层 | Spring Boot、Spring Security、MyBatis Plus、若依框架 | RESTful API接口提供、业务逻辑处理、安全访问控制、系统监控数据采集、安全检测引擎实现 |
   | 数据存储层 | MySQL、Redis                                   | 业务数据持久化存储、缓存数据管理、日志与安全事件数据存储                    |
2. **核心功能模块设计**
   项目基于需求拆解为6大基础模块+1个核心安全模块，其中安全态势感知模块为项目核心创新模块，包含安全仪表盘、安全事件管理、安全规则管理、智能检测引擎、自动化响应机制、安全评分预警6个子模块，实现了安全威胁的全生命周期管理。
3. **数据库设计**
   项目基于若依框架原生数据库结构，扩展了安全态势感知核心业务表，核心包括：安全事件表、安全规则表、安全评分历史表，同时复用系统原生的系统访问记录表、操作日志记录表，完成了全业务数据的结构化存储，表结构设计符合数据库范式，字段定义清晰，满足业务查询与审计需求。

### （三）系统实现与测试

1. **系统实现核心内容**
   - 开发环境：基于OpenEuler 24.03 Standard ISO版本最小化安装，通过dnf包管理器按需搭建Java、Node.js、MySQL、Git开发环境，兼顾轻量化与开发需求；
   - 基础功能实现：复用若依框架自带的监控接口，实现系统监控、缓存管理、定时任务、日志审计等基础功能，避免重复开发，大幅缩短开发周期；
   - 可视化实现：基于ECharts组件，通过仪表盘、饼图、折线图、柱状图等多种图表类型，实现CPU/内存使用率、安全评分、事件分布等数据的可视化展示；
   - 安全核心能力实现：开发了四大智能检测引擎（登录检测、网络检测、文件检测、进程检测），配套五大自动化响应动作（IP封禁、进程终止、连接阻断、文件恢复、无响应），同时实现了动态安全评分系统，完成了安全闭环的落地。
2. **系统测试与结果**
   项目在1核CPU、4GB内存、40GB磁盘的OpenEuler 24.03虚拟机环境中完成测试，核心针对安全态势感知四大核心功能设计了13个测试用例，测试结果如下：
   - 登录检测与IP封禁功能：4个测试用例全部通过，可准确识别暴力破解攻击并自动执行IP封禁；
   - 文件检测功能：3个测试用例全部通过，可精准识别敏感文件异常操作并触发文件恢复响应；
   - 网络检测功能：3个测试用例全部通过，可有效识别异常网络连接并执行连接阻断；
   - 进程检测功能：3个测试用例全部通过，可准确识别恶意/未授权进程并执行进程终止；
   - 整体测试结论：13个测试用例通过率100%，检测准确率100%，威胁响应时间＜1秒，各项功能均达到设计要求，同时轻量化性能指标全部达标。

## 四、项目核心优势与创新点

1. **国产化深度适配，贴合信创产业趋势**
   项目以国产开源操作系统OpenEuler为核心基座，完成了全栈技术在国产操作系统环境下的适配与验证，填补了信创环境下轻量级服务器安全监控方案的实践空白，顺应了政企数字化转型中国产化替代的核心需求，具备较强的行业适配性。
2. **极致轻量化设计，适配多元场景**
   项目通过最小化系统安装、按需配置依赖、复用成熟框架接口，实现了极低的资源占用，在1核CPU、4GB内存的资源受限环境中稳定运行，CPU占用率低于10%，内存占用低于200MB，不仅适配常规服务器环境，也可满足边缘计算、物联网等轻量化场景的监控需求，场景适配性远高于传统监控工具。
3. **创新的动态安全评分体系，实现安全态势量化呈现**
   项目设计了0-100分的动态安全评分系统，综合安全事件的数量与级别、时间分布权重、处理状态、安全规则启用情况四大维度，通过加权算法实时计算系统安全状态，近期事件权重更高、已处理事件影响更低，同时配套评分趋势图表与阈值预警机制，将复杂的网络安全态势转化为直观可理解的量化指标，大幅降低了安全运维的门槛。
4. **完整的检测-响应安全闭环，自动化处置能力完善**
   项目突破了传统监控工具“只监不防”的局限，搭建了四大智能检测引擎+五大自动化响应动作的完整安全闭环，可实时识别暴力破解、敏感文件篡改、异常网络访问、恶意进程运行等常见安全威胁，并在1秒内自动执行对应的处置动作，实现了从威胁发现到风险处置的全流程自动化，有效提升了服务器安全防护能力。
5. **低开发成本与高可用性兼顾，落地性强**
   项目通过复用若依框架的成熟功能与原生接口，避免了从零开发的高额成本，大幅缩短了开发周期，同时基于前后端分离架构打造了友好的可视化界面，操作便捷、学习门槛低，既解决了传统命令行监控工具使用难度高的问题，也规避了商业化监控软件授权成本高昂的痛点，尤其适配中小企业、个人开发者与教育场景的使用需求。

## 五、项目存在的不足与局限性

结合项目设计实现与测试验证内容，项目仍存在以下可优化与待完善的方向，核心局限如下：

1. **威胁检测能力深度不足**
   目前系统仅实现了基于预设规则与阈值的基础检测能力，对APT攻击、零日漏洞攻击、内部威胁等复杂攻击场景的检测能力缺失，检测引擎缺乏基于行为分析的智能识别能力，对未知威胁的适应性不足。
2. **威胁情报与外部数据融合能力缺失**
   系统仅依赖本地检测规则实现安全监控，未对接外部威胁情报平台，无法实时获取最新的恶意IP、恶意域名、攻击指标等情报数据，难以应对快速迭代的新型网络攻击，主动防御与预警能力受限。
3. **数据分析与挖掘能力薄弱**
   系统仅实现了安全评分与历史趋势的基础展示，对历史安全事件的深度分析能力不足，缺乏安全事件关联分析、攻击链重构、根因分析等高级功能，无法从海量数据中提取高价值的安全情报，难以支撑深度安全决策。
4. **核心配套机制不完善**
   告警通知机制仅支持系统内弹窗预警，缺乏邮件、短信、微信、钉钉等多渠道通知能力，无人值守场景下的实时告警能力不足；同时系统仅支持单台服务器监控，缺乏多服务器集群的集中管理与统一监控能力，企业级大规模应用场景适配性不足。
5. **性能与终端适配性有待优化**
   系统虽满足基础轻量化需求，但缺乏对海量日志、高并发场景的性能优化，随着监控指标与检测规则的增加，资源占用与响应时间可能出现显著增长；同时界面仅针对PC浏览器设计，移动端响应式布局缺失，移动设备访问体验较差。

### （一）核心应用价值

1. **理论价值**
   项目探索了基于OpenEuler国产操作系统与若依开源框架的安全态势感知实现方案，丰富了国产操作系统应用生态的研究内容，拓展了若依框架在系统监控与网络安全领域的应用场景，为同类型课题研究提供了可参考的思路与实践方案。
2. **实践价值**
   项目实现了系统监控与安全防护能力的一体化落地，大幅降低了服务器运维与安全管理的成本与门槛，可直接应用于OpenEuler环境下的服务器运维工作，为中小企业、个人开发者、教育机构提供了一套开源免费、轻量化、易部署的服务器安全监控解决方案。
3. **教学价值**
   项目完整覆盖了从国产操作系统环境搭建、前后端开发、安全引擎设计、系统测试全流程，技术栈贴合高校网络安全、软件工程专业的教学内容，可作为操作系统、网络安全、Web开发等课程的实践教学案例，具备较强的教育推广价值。

### （二）应用场景

1. 中小企业信创环境下的应用服务器运维与安全监控；
2. 边缘计算、物联网网关等资源受限场景的轻量化安全监控；

随着国内信创产业的持续深化，OpenEuler等国产操作系统的部署规模将持续扩大，政企、中小企业对国产化环境下的安全监控与态势感知解决方案的需求将持续增长。本项目作为贴合国产化趋势的轻量化开源方案，若能针对现有不足完成优化迭代，引入机器学习、威胁情报融合、多节点集群管理等能力，可进一步拓展应用边界，在国产操作系统安全生态中具备较强的发展潜力。

## 七、项目整体总结

本项目完整完成了“需求分析-架构设计-系统实现-测试验证”的全流程研发，基于OpenEuler国产操作系统成功实现了一套集系统监控、可视化展示、安全威胁检测、自动化响应于一体的轻量化安全态势感知系统。项目核心功能全部通过测试验证，各项性能指标达到设计目标，解决了传统监控工具的核心痛点，贴合国产化信创发展趋势，具备较强的实践应用价值与研究参考意义。
