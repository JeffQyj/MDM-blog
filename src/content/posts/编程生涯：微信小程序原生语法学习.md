## 引言：从 Web 到小程序

2024 年 8 月中旬，我开始系统学习微信小程序开发。

在这之前，我已经掌握了 HTML、CSS、JavaScript 前端三件套，也写过一些粗糙的网页Demo。自以为「前端不过如此」的我，在真正接触小程序时才发现——**原来「类 Web」和「真 Web」之间，隔着一整套全新的技术范式。**

小程序不是简单地把 HTML 换成 WXML、把 CSS 换成 WXSS。它有自己独特的事件系统、生命周期、双线程架构，甚至对 JavaScript 的使用方式也有平台特定的约束。

这篇文章，就来完整记录我学习小程序原生语法的全过程。

## 小程序与前端的核心差异

### 技术架构对比

在动手写代码之前，必须先理解小程序的运行原理。与传统 Web 前端相比，小程序最大的区别在于**双线程架构**：

| 层级 | Web 前端 | 微信小程序 |
|------|---------|-----------|
| 结构层 | HTML | WXML |
| 样式层 | CSS | WXSS |
| 逻辑层 | JavaScript | JavaScript |
| 渲染引擎 | 浏览器 WebView | 微信客户端渲染引擎 |
| 架构特点 | 单线程（DOM 操作） | 双线程（逻辑层 + 视图层分离） |

**双线程架构的工作原理**：小程序的视图层和逻辑层分别运行在不同的线程中。逻辑层负责处理数据、业务逻辑、调用 API；视图层负责渲染页面、响应用户交互。两者之间通过 `setData` 进行异步通信。

这种设计的优势在于：**避免了直接操作 DOM 带来的性能问题**，渲染效率和稳定性都有显著提升。但代价是开发者需要适应「数据驱动视图」而非「DOM 操作视图」的编程范式。

### 语法对照速查表

| 功能 | Web 语法 | 小程序语法 |
|------|---------|-----------|
| 页面结构 | `<div>`、`<span>` | `<view>`、`<text>` |
| 数据绑定 | 无（手动操作 DOM） | `{{variable}}` |
| 条件渲染 | JavaScript `if` | `wx:if` / `wx:elif` / `wx:else` |
| 列表渲染 | `array.map()` | `wx:for` |
| 事件绑定 | `onclick=""` | `bindtap=""` / `catchtap=""` |
| 数据更新 | `element.innerText = ''` | `this.setData({})` |
| 样式导入 | `@import url()` | `@import "path"` |

## 项目结构：约定式开发的魅力

小程序采用**约定式项目结构**，每个页面都遵循固定的文件组织方式：

```
├── app.js              # 应用入口，全局逻辑
├── app.json            # 全局配置（页面路由、窗口样式、tabBar）
├── app.wxss            # 全局样式
├── project.config.json # 项目配置文件（IDE 相关）
├── sitemap.json        # 微信搜索配置
├── pages/              # 页面目录
│   └── index/          # 某个页面
│       ├── index.js    # 页面逻辑
│       ├── index.wxml  # 页面结构
│       ├── index.wxss # 页面样式
│       └── index.json  # 页面配置
├── components/         # 自定义组件目录
└── utils/              # 工具函数
```

:::note
全局配置中的 `pages` 数组定义了小程序的所有页面，**第一个页面是默认首页**。新增页面时需要手动添加到这个数组中，否则页面无法访问。
:::

### 全局配置 app.json

```json
{
  // ==================== 页面路由配置 ====================
  // pages 数组定义了小程序的所有页面路径
  // 数组中的第一个路径是默认首页，小程序启动时会首先加载这个页面
  // 新增页面时必须先在此添加路径，否则页面无法访问
  "pages": [
    "pages/index/index",    // 首页路径（默认启动页）
    "pages/logs/logs",    // 日志页面
    "pages/user/user"      // 用户页面
  ],

  // ==================== 窗口样式配置 ====================
  // window 配置项用于设置小程序的导航栏、背景色等窗口样式
  "window": {
    // 导航栏标题文字（每个页面可通过 page.json 单独覆盖）
    "navigationBarTitleText": "我的小程序",

    // 导航栏背景颜色，使用十六进制颜色值
    "navigationBarBackgroundColor": "#ffffff",

    // 导航栏文字颜色，仅支持 "black"（黑色）或 "white"（白色）
    "navigationBarTextStyle": "black",

    // 页面背景颜色（即可下拉刷新时的背景色）
    "backgroundColor": "#f5f5f5",

    // 是否开启全局下拉刷新功能
    // true：开启；false：关闭（页面级别可通过 page.json 单独控制）
    "enablePullDownRefresh": false
  },

  // ==================== 底部 TabBar 配置 ====================
  // tabBar 用于配置小程序底部的导航栏，通常用于主页面切换
  "tabBar": {
    // 默认文字颜色（未选中状态）
    "color": "#999999",

    // 选中时的文字颜色
    "selectedColor": "#1890ff",

    // TabBar 背景颜色
    "backgroundColor": "#ffffff",

    // TabBar 列表，每个项对应一个页面（至少 2 个，最多 5 个）
    "list": [
      {
        // 页面路径，必须在 pages 数组中定义过
        "pagePath": "pages/index/index",

        // TabBar 显示的文案
        "text": "首页"
      },
      {
        "pagePath": "pages/user/user",
        "text": "我的"
      }
    ]
  },

  // ==================== 网络请求超时配置 ====================
  // 设置各种网络请求的超时时间，单位为毫秒（ms）
  "networkTimeout": {
    "request": 10000,        // wx.request 请求超时时间
    "connectSocket": 10000,  // WebSocket 连接超时时间
    "uploadFile": 10000,     // 文件上传超时时间
    "downloadFile": 10000    // 文件下载超时时间
  },

  // ==================== 代码加载优化 ====================
  // lazyCodeLoading 用于配置代码的按需加载模式
  // "requiredComponents"：只注入当前页面需要的自定义组件，可显著降低启动时间
  "lazyCodeLoading": "requiredComponents"
}
```

### 页面级配置 page.json

页面目录下的 `page.json` 用于覆盖全局配置，实现页面级别的个性化设置：

```json
{
  // 单独设置当前页面的导航栏标题
  // 会覆盖 app.json 中 window.navigationBarTitleText 的值
  "navigationBarTitleText": "详情页",

  // 当前页面开启下拉刷新功能
  // 可在页面滚动时下拉触发 onPullDownRefresh 生命周期
  "enablePullDownRefresh": true,

  // 页面上拉触底距离，单位为 px
  // 设置后，页面滚动到底部指定距离时会触发 onReachBottom
  // 通常用于实现"加载更多"功能
  "onReachBottomDistance": 50
}
```

## WXML 模板语法：数据驱动的视图构建

WXML（WeiXin Markup Language）是小程序的页面结构描述语言，语法与 HTML 类似，但增加了很多**模板语法和数据绑定**能力。

### 数据绑定

数据绑定是小程序最核心的概念之一。页面 `data` 中定义的数据，会在 WXML 中通过 `{{}}` 语法进行绑定：

```javascript
// index.js - 页面逻辑文件
// 使用 Page() 构造器注册一个页面
Page({

  // ========== data 对象：页面初始数据的定义 ==========
  // data 中的数据会在对应的 WXML 模板中通过 {{}} 语法展示
  // 所有在 WXML 中使用的数据都必须先在 data 中定义
  data: {
    // 字符串类型数据
    message: 'Hello World',

    // 数字类型数据
    count: 0,

    // 布尔类型数据（常用于控制显示/隐藏）
    isLoading: true,

    // 对象类型数据（可以嵌套任意深度）
    userInfo: {
      name: '小明',  // 用户姓名
      age: 18        // 用户年龄
    }
  }

  // ========== 注意：不要在 data 外面定义页面状态 ==========
  // 页面的响应式数据必须放在 data 对象内部
  // 在 WXML 中直接访问 this.data.xxx 即可获取数据
})
```

```xml
<!-- index.wxml - 页面结构文件 -->
<!-- 通过 {{}} 语法将 JS 中定义的数据渲染到页面上 -->

<!-- 基础文本绑定：直接使用双花括号包裹变量名 -->
<view>{{message}}</view>                    <!-- 输出：Hello World -->

<!-- 表达式绑定：花括号内可以进行运算、拼接等操作 -->
<view>计数：{{count}}</view>                <!-- 输出：计数：0 -->
<view>年龄：{{userInfo.age + 1}}</view>      <!-- 输出：年龄：19 -->

<!-- 条件绑定：布尔值配合 wx:if 控制元素显示 -->
<view wx:if="{{isLoading}}">加载中...</view>  <!-- isLoading=true 时显示 -->

<!-- 对象属性绑定：通过点语法访问嵌套对象的属性 -->
<view>姓名：{{userInfo.name}}</view>         <!-- 输出：姓名：小明 -->
```

:::tip
`setData` 是更新数据的唯一方式。**不要尝试直接修改 `this.data`**——这不会触发视图更新。
:::

```javascript
// ==================== 数据更新方式对比 ====================

// ❌ 错误做法：直接修改 data（不会触发视图更新）
// 这种写法虽然不会报错，但页面不会重新渲染
this.data.count = 10;

// ✅ 正确做法：通过 setData 方法更新数据
// setData 会将数据从逻辑层发送到视图层，触发页面重新渲染
this.setData({
  count: 10  // key 是 data 中的属性名，value 是新的值
});

// setData 也支持同时更新多个数据
this.setData({
  count: 10,
  message: '更新后的消息',
  'userInfo.name': '小红'  // 可以通过字符串路径更新嵌套属性
});
```

### 条件渲染

条件渲染用于根据数据状态动态显示/隐藏元素，类似于 JavaScript 中的 if/else 逻辑：

```xml
<!-- ==================== wx:if 条件渲染家族 ==================== -->

<!-- wx:if：条件为 true 时渲染元素，为 false 时完全不渲染 -->
<!-- wx:elif：第一个条件不满足时的备选条件 -->
<!-- wx:else：所有条件都不满足时渲染 -->

<!-- 示例：根据分数显示不同评级 -->
<view wx:if="{{score >= 90}}">优秀</view>                    <!-- 90分以上显示 -->
<view wx:elif="{{score >= 60}}">及格</view>                   <!-- 60-89分显示 -->
<view wx:else>不及格</view>                                     <!-- 60分以下显示 -->


<!-- ==================== hidden 与 wx:if 的区别 ==================== -->

<!-- hidden：通过 CSS display 属性控制显示/隐藏 -->
<!--       元素仍然存在于 DOM 中，只是视觉上不可见 -->
<view hidden="{{isHidden}}">通过样式隐藏</view>

<!-- wx:if：通过增删 DOM 节点控制显示/隐藏 -->
<!--        条件为 false 时，元素会从 DOM 中完全移除 -->
<view wx:if="{{!isHidden}}">直接移除元素</view>


<!-- ==================== 完整示例 ==================== -->
<!-- 示例：根据登录状态显示不同内容 -->
<view wx:if="{{isLoggedIn}}">
  <text>欢迎回来，{{userName}}</text>
  <button bindtap="logout">退出登录</button>
</view>
<view wx:else>
  <button bindtap="login">登录</button>
  <button bindtap="register">注册</button>
</view>
```

| 属性 | wx:if | hidden |
|------|-------|--------|
| 实现方式 | 添加/移除 DOM | 添加 `display: none` 样式 |
| 性能消耗 | 高（涉及 DOM 操作） | 低（仅切换样式） |
| 适用场景 | 条件不频繁切换 | 条件频繁切换 |

### 列表渲染

列表渲染用于循环生成多个元素，类似于 JavaScript 中的 map() 或 forEach()：

```xml
<!-- ==================== 基础数组循环 ==================== -->

<!--
  wx:for：循环遍历数组，item 是当前项，index 是当前索引
  wx:key：指定列表中每个元素的唯一标识，用于优化渲染性能
-->
<view wx:for="{{items}}" wx:key="index">
  {{index + 1}}. {{item}}
</view>

<!-- 假设 items = ['苹果', '香蕉', '橘子'] -->
<!-- 渲染结果：1. 苹果  2. 香蕉  3. 橘子 -->


<!-- ==================== 自定义循环项和索引名称 ==================== -->

<!--
  wx:for-item：指定当前循环项的变量名（默认是 item）
  wx:for-index：指定当前索引的变量名（默认是 index）
-->
<view wx:for="{{users}}" wx:for-item="user" wx:for-index="i" wx:key="id">
  {{i + 1}}. {{user.name}} - {{user.age}}岁
</view>

<!-- 假设 users = [{id: 1, name: '小明', age: 18}, {id: 2, name: '小红', age: 20}] -->
<!-- 渲染结果：1. 小明 - 18岁  2. 小红 - 20岁 -->


<!-- ==================== wx:key 的取值方式 ==================== -->

<!-- 方式1：字符串 - 使用对象中的某个唯一属性作为标识 -->
<view wx:for="{{users}}" wx:key="id">{{item.name}}</view>      <!-- id 是对象的唯一属性 -->

<!-- 方式2：*this - 当数组是普通类型数组时使用，表示循环项本身 -->
<view wx:for="{{[1, 2, 3, 4, 5]}}" wx:key="*this">{{item}}</view>
<!-- 注意：只有数组元素唯一时才能用 *this -->


<!-- ==================== 对象循环 ==================== -->

<!--
  遍历对象时：
  wx:for-item：对象的值（默认是 item）
  wx:for-index：对象的键（默认是 index）
-->
<view wx:for="{{obj}}" wx:for-item="value" wx:for-index="key" wx:key="key">
  {{key}}: {{value}}
</view>

<!-- 假设 obj = {name: '小明', age: 18, city: '北京'} -->
<!-- 渲染结果：name: 小明  age: 18  city: 北京 -->


<!-- ==================== 嵌套循环的注意事项 ==================== -->

<!-- 多层循环时，必须使用不同的循环项和索引名，避免命名冲突 -->
<view wx:for="{{categories}}" wx:for-item="category" wx:key="id">
  <text>{{category.name}}</text>
  <view wx:for="{{category.products}}" wx:for-item="product" wx:for-index="pIndex" wx:key="id">
    - {{product.name}}
  </view>
</view>
```

:::note
`wx:key` 用于指定列表中元素的唯一标识，可以是字符串（对象属性名）或 `*this`（表示循环项本身）。正确使用 `wx:key` 能显著提升列表渲染性能。
:::

### 模板引用

WXML 支持两种模板引用方式，用于复用 WXML 代码片段：

```xml
<!-- ==================== import：引入模板 ==================== -->
<!-- import 用于引入定义好的模板文件 -->
<!-- 特点：只会引入模板内容，不会执行任何脚本逻辑 -->
<!-- 注意：import 具有作用域，只能引入外层模板，外层模板中的 import 不会被传递 -->

<!-- 语法：src 属性指定模板文件路径 -->
<import src="./template.wxml" />


<!-- ==================== include：引入整个模板 ==================== -->
<!-- include 用于引入整个 WXML 文件的内容（相当于复制粘贴） -->
<!-- 特点：会引入目标文件的全部内容，但 <template> 标签除外 -->
<!-- 常用于引入公共页面头部、底部等结构 -->

<!-- 语法：src 属性指定要引入的文件路径 -->
<include src="./header.wxml" />


<!-- ==================== 模板定义 ==================== -->
<!-- <template> 标签用于定义可复用的模板 -->
<!-- name 属性是模板的唯一标识，在引用时通过 is 属性指定 -->

<!-- 定义一个用户卡片模板 -->
<template name="userCard">
  <!-- 模板内部可以使用传入的任意数据 -->
  <view class="card">
    <text class="name">{{name}}</text>
    <text class="desc">{{desc}}</text>
  </view>
</template>


<!-- ==================== 模板使用 ==================== -->
<!--
  is 属性：指定要使用的模板名称
  data 属性：传递给模板的数据，使用 ... 展开运算符传入整个对象
-->
<template is="userCard" data="{{...userData}}" />

<!-- 假设 userData = {name: '小明', desc: '这是一个用户描述'} -->
<!-- 模板会渲染出：<view class="card"><text>小明</text><text>这是一个用户描述</text></view> -->


<!-- ==================== 完整示例 ==================== -->

<!-- header.wxml - 头部模板 -->
<template name="header">
  <view class="header">
    <text class="title">{{title}}</text>
    <text class="subtitle">{{subtitle}}</text>
  </view>
</template>

<!-- page.wxml - 页面中使用 -->
<template name="header" src="./header.wxml"></template>

<!-- 传入不同数据复用同一模板 -->
<template is="header" data="{{{title: '首页', subtitle: '欢迎来到首页'}}" />
<template is="header" data="{{{title: '分类', subtitle: '浏览商品分类'}}" />
```

## WXSS 样式：移动端适配利器

WXSS（WeiXin Style Sheets）是微信小程序的样式语言，与 CSS 基本兼容，并做了一些移动端适配增强。

### rpx 响应式单位

rpx（responsive pixel）是小程序特有的尺寸单位，**以屏幕宽度为基准**：750rpx = 屏幕宽度。

这意味着在不同尺寸的设备上，元素的相对大小保持一致：

| 设备宽度 | 350rpx 实际宽度 |
|---------|----------------|
| iPhone 6 (375px) | 175px |
| iPhone 12 (390px) | 182px |
| iPad (768px) | 384px |

```css
/* ==================== rpx 单位的特点 ==================== */

/*
 * 设计稿通常以 750px 宽度为基准（iPhone 6 的屏幕宽度）
 * 使用 rpx 单位可以自动适配不同屏幕尺寸
 * 换算公式：元素宽度(px) = 设计稿宽度 / 2
 */

/* 典型用法：全屏宽度卡片 */
/* 750rpx = 屏幕宽度，所以 710rpx 意味着左右各留 20rpx 的边距 */
.card {
  width: 710rpx;      /* 元素宽度：占屏幕宽度的 94.67% */
  padding: 20rpx;     /* 内边距：左右各 20rpx */
  margin: 20rpx auto; /* 上下 20rpx，左右 auto 居中 */
}

/* 按钮尺寸：固定高度和宽度 */
.btn {
  width: 200rpx;       /* 按钮宽度 */
  height: 80rpx;       /* 按钮高度 */
  line-height: 80rpx;   /* 文字垂直居中 */
  text-align: center;   /* 文字水平居中 */
  border-radius: 40rpx; /* 圆角半径为宽度的1/2，呈现胶囊形状 */
}

/* 字体大小也推荐使用 rpx，以保持各设备上视觉一致性 */
.title {
  font-size: 32rpx;     /* 标题字体 */
}
.content {
  font-size: 28rpx;     /* 正文字体 */
}
.caption {
  font-size: 24rpx;     /* 辅助说明文字 */
}
```

### calc 表达式

WXSS 支持 `calc()` 表达式，但需要注意**运算符两边必须留空格**：

```css
/* ==================== calc 注意事项 ==================== */

/* ❌ 错误写法：运算符紧邻数字，会导致解析失败 */
width: calc(750rpx-40rpx);        /* 注意：- 两边没有空格 */
height: calc(100rpx+50rpx);       /* 注意：+ 两边没有空格 */

/* ✅ 正确写法：运算符两边各留一个空格 */
width: calc(750rpx - 40rpx);      /* 正确：750rpx - 40rpx = 710rpx */
height: calc(100rpx + 50rpx);      /* 正确：100rpx + 50rpx = 150rpx */

/* calc 表达式的典型应用场景 */
.container {
  width: calc(750rpx - 40rpx);    /* 总宽度减去左右边距 */
  height: calc(100vh - 120rpx);   /* 视口高度减去头部和底部高度 */
  margin: calc(20rpx / 2);        /* 使用除法（结果为 10rpx） */
}
```

### 样式导入


### 模块化

小程序支持 CommonJS 规范的模块化，允许将公共代码提取到单独文件中复用：

```javascript

/*
 * module.exports：向外暴露模块的公共接口
 * 被暴露的属性和方法可以被其他文件通过 require() 引入后使用
 */

// 定义一个加法函数
function add(a, b) {
  return a + b;
}

// 定义一个乘法函数
function multiply(a, b) {
  return a * b;
}

// 向外暴露接口（可以暴露对象，也可以直接暴露函数）
module.exports = {
  add: add,      // 属性名: 函数名（可简写为 add）
  multiply: multiply
};

// ==================== 其他页面中使用工具模块 ====================

// 通过 require() 引入工具模块
// 路径可以是相对路径（./ 或 ../）或绝对路径（/ 开头）
var math = require('../../utils/math.js');

// ==================== ES6 Module 语法（小游戏中使用） ====================
// 注意：微信小程序逻辑层主要使用 CommonJS，ES6 Module 在部分场景下可用

// export.js
export const name = '小明';

  console.log('Hello');
}
```

### Page 页面构造器

每个页面都需要在 JS 文件中使用 `Page()` 构造器注册，Page 是小程序页面的入口函数：

```javascript
// pages/index/index.js - 页面逻辑文件
// 使用 Page() 构造器注册一个页面，页面名称必须与目录名对应

Page({

  // ==================== data：页面初始数据 ====================
  // data 中的数据会在对应的 WXML 模板中通过 {{}} 语法展示
  // 页面加载时会自动使用 data 中的数据渲染页面
  data: {
    title: '首页',   // 页面标题
    list: []         // 列表数据（通常从接口获取）
  },


  // ==================== onLoad：页面加载时触发 ====================
  // 页面加载时自动调用，整个页面生命周期只调用一次
  // options：页面跳转时携带的参数（通过 url?key=value&key2=value2 传递）
  onLoad: function(options) {
    console.log('页面加载', options);

    // 典型用法：从 URL 参数中获取数据
    // 例如：pages/detail/detail?id=1 跳转后，options = {id: '1'}
    const articleId = options.id;

    // 常在此处发起网络请求获取初始数据
    this.fetchArticleList();
  },

  // 每次页面从隐藏切换到显示状态时都会调用
  // 适用场景：页面切换回来时刷新最新数据
  onShow: function() {
    console.log('页面显示');

    // 场景：用户从其他页面返回时刷新数据
    // this.fetchArticleList();
  },


  // ==================== onReady：页面初次渲染完成时触发 ====================
  // 页面第一次渲染完成后调用，整个生命周期只调用一次
  // 此时页面已经可以与用户进行交互
  // 适用场景：获取页面元素信息、启动动画等
  onReady: function() {
    console.log('页面渲染完成');

    // 典型用法：获取页面元素尺寸（必须等渲染完成才能获取）

  // 适用场景：暂停音乐、停止动画、保存页面状态
  onHide: function() {
    console.log('页面隐藏');

    // 典型用法：暂停播放中的视频或音频

  // 适用场景：清理定时器、取消订阅、解绑事件监听
  onUnload: function() {
    console.log('页面卸载');

    // 典型用法：清理在 onLoad 中启动的定时器
    // clearInterval(this.data.timer);

  // 开启下拉刷新功能：需要在 page.json 中设置 "enablePullDownRefresh": true
  // 用户下拉页面时自动触发
  onPullDownRefresh: function() {
    console.log('下拉刷新');

    // 典型用法：重新请求接口获取最新数据
    this.fetchArticleList().then(() => {
      // 数据加载完成后，必须手动停止下拉刷新动画
      wx.stopPullDownRefresh();

  },

  // 用户滚动到底部时自动触发
  // 适用场景：实现"加载更多"功能

  onReachBottom: function() {
    console.log('触底加载更多');

    // 典型用法：加载下一页数据
    // this.setData({ page: this.data.page + 1 });
    // this.fetchArticleList(true);
  },


  // ==================== onPageScroll：页面滚动时触发 ====================
  // e.scrollTop：当前页面滚动过的垂直距离（单位：px）
  // 注意：此回调函数会被频繁调用，避免在内部执行复杂操作
  onPageScroll: function(e) {
    console.log('滚动位置', e.scrollTop);

    // 典型用法：实现吸顶导航栏效果
    // if (e.scrollTop > 100) { this.setData({ fixed: true }); }
  },


  // ==================== 自定义页面方法 ====================
  // 在 methods 对象中定义页面的事件处理函数和自定义方法
  methods: {

    // 点击事件处理函数
    handleTap: function(e) {
      console.log('点击事件', e);

      // 更新页面数据（必须使用 setData）
      this.setData({
        title: '点击后的标题'
      });
    },

    // 网络请求示例方法
    fetchArticleList: function(isAppend = false) {

            that.setData({
              list: that.data.list.concat(res.data.list)
            });
          } else {
            // 替换数据：直接设置新数据
            that.setData({ list: res.data.list });
          }
        }
      });
    }
  }
});
```

### Component 组件构造器

自定义组件使用 `Component()` 构造器注册，与 Page 构造器类似但功能更强大：

```javascript
// components/custom-card/custom-card.js - 自定义卡片组件

/*
 * Component 构造器用于创建自定义组件
 * 组件是独立的可复用单元，有自己的模板、样式和逻辑
 */

Component({

  // ==================== properties：组件属性（对外接口） ====================
  // properties 用于接收外部传入的数据，类似于 React 的 props
  // 定义了组件可以被外部配置的属性及其类型、默认值
  properties: {
    // 简写方式：type 是必填项，value 是默认值
    title: {
      type: String,           // 属性类型：String（字符串）
      value: '默认标题'       // 默认值：当外部没有传入时使用此值
    },

    // 完整写法：可以设置更多选项
    content: {
      type: String,           // 属性类型
      value: ''               // 默认空字符串
    },

    // 支持多种类型
    count: {
      type: Number,          // 数字类型
      value: 0
    },
    isShow: {
      type: Boolean,         // 布尔类型
      value: false
    },
    userInfo: {
      type: Object,          // 对象类型
      value: null
    },
    tags: {
      type: Array,           // 数组类型
      value: []
    }
  },


  // ==================== data：组件内部数据 ====================
  // data 与 properties 的区别：
  // - properties：对外数据（由外部传入）
  // - data：内部数据（组件自己管理）
  data: {
    isExpanded: false,        // 展开/收起状态
    internalCount: 0          // 内部计数器
  },


  // ==================== methods：组件方法 ====================
  // 组件的事件处理函数和自定义方法必须定义在 methods 中
  methods: {

    // 点击卡片头部触发展开/收起
    toggleExpand: function() {
      // 读取内部状态
      const currentState = this.data.isExpanded;

      // 更新内部状态（必须使用 setData）
      this.setData({
        isExpanded: !currentState
      });

      // 向外部触发自定义事件（通知父组件状态变化）
      // triggerEvent(事件名, 传递的数据, 事件选项)
      this.triggerEvent('toggle', {
        expanded: !currentState,
        title: this.data.title
      });
    },

    // 处理按钮点击
    onButtonTap: function() {
      // 获取外部传入的属性值
      console.log('按钮点击', this.properties.title);

      // 触发自定义事件
      this.triggerEvent('click');
    },

    // 外部调用方法（供父组件通过 selectComponent 调用）
    getExpandedState: function() {
      return this.data.isExpanded;
    }
  },


  // ==================== lifetimes：组件生命周期 ====================
  // lifetimes 用于定义组件自身的生命周期钩子
  lifetimes: {

    // created：组件实例创建时调用
    // 此时还不能使用 setData、获取节点信息
    // 适用：初始化组件内部状态
    created: function() {
      console.log('组件实例创建');
      // this.data.xxx 可以访问，但不能 this.setData()
    },

    // attached：组件添加到页面时调用
    // 此时组件已经可以访问 properties 和 data
    // 适用：获取组件配置、初始化渲染
    attached: function() {
      console.log('组件添加到页面');
      // 可以使用 this.data 和 this.setData()
    },

    // ready：组件布局完成时调用
    // 此时可以获取到节点信息（尺寸、位置等）
    // 适用：执行依赖 DOM 布局的操作
    ready: function() {
      console.log('组件布局完成');

      // 获取组件的尺寸信息
      // wx.createSelectorQuery().in(this).select('.card').boundingClientRect();
    },

    // detached：组件从页面移除时调用
    // 适用：清理定时器、取消事件订阅、释放资源
    detached: function() {
      console.log('组件从页面移除');

      // 清理定时器
      // clearInterval(this.data.timer);

      // 取消事件订阅
      // wx.offSocketOpen();
    }
  },


  // ==================== pageLifetimes：组件所在页面的生命周期 ====================
  // 用于监听组件所在页面的生命周期变化
  pageLifetimes: {

    // show：页面显示时调用
    show: function() {
      console.log('组件所在页面显示');
    },

    // hide：页面隐藏时调用
    hide: function() {
      console.log('组件所在页面隐藏');
    }
  }
});
```

### 组件生命周期详解

| 生命周期 | 触发时机 | 适用场景 |
|---------|---------|---------|
| created | 组件实例创建 | 初始化内部状态 |
| attached | 组件添加到页面 | 获取节点信息、初始化渲染 |
| ready | 组件布局完成 | 执行依赖 DOM 的操作 |
| moved | 组件被移动 | 较少使用 |
| detached | 组件从页面移除 | 清理定时器、取消订阅 |

## 事件系统：用户交互的核心

小程序的事件系统与 Web DOM 事件有显著区别。

### 事件绑定

小程序的事件系统与 Web DOM 事件有显著区别，采用的是一种「bind/catch + 事件名」的绑定方式：

```xml
<!-- ==================== bind：普通绑定（支持冒泡） ==================== -->
<!-- bind + 事件名：绑定事件，支持事件向上冒泡传播 -->
<!-- 适用场景：大部分需要响应用户操作的场景 -->

<!-- 点击事件：手指轻触屏幕后抬起触发 -->
<view bindtap="handleTap">点击我（会冒泡）</view>

<!-- 长按事件：手指触摸超过 300ms 触发 -->
<view bindlongpress="handleLongPress">长按我</view>


<!-- ==================== catch：阻止冒泡绑定 ==================== -->
<!-- catch + 事件名：绑定事件，但阻止事件向上冒泡传播 -->
<!-- 适用场景：需要阻止事件传播给父元素的场景 -->

<!-- 点击事件：点击后不会向上冒泡 -->
<view catchtap="handleCatch">阻止冒泡</view>


<!-- ==================== capture-bind/capture-catch：捕获阶段绑定 ==================== -->
<!-- 事件传播分为「捕获阶段」和「冒泡阶段」 -->
<!-- capture-bind：捕获阶段绑定（从外到内传递） -->
<!-- capture-catch：捕获阶段绑定并阻止后续传播 -->

<view capture-bind:tap="handleCapture">
  <text>capture-bind 会先触发</text>
</view>


<!-- ==================== 组件自定义事件 ==================== -->
<!-- 组件通过 triggerEvent 触发的事件，需要用 bind: 来监听 -->

<!-- my-component 是自定义组件 -->
<!-- bind:custom 是组件触发的自定义事件名 -->
<!-- onCustomEvent 是处理函数 -->
<custom-component bind:custom="onCustomEvent" />

<!-- 监听带命名空间的自定义事件 -->
<custom-component bind:myNamespace:myevent="onMyEvent" />
```


### 事件对象

事件处理函数会接收一个事件对象 `e`，包含丰富的信息：

```javascript
// 事件处理函数定义在 Page 或 Component 的 methods 中
methods: {

  // 所有事件处理函数都会接收一个事件对象参数
  // 通常命名为 e 或 event
  handleTap: function(e) {

    // ==================== 事件基本信息 ====================

    // type：事件类型，如 "tap"、"longpress"
    console.log('事件类型', e.type);

    // timeStamp：事件触发的时间戳（页面打开到现在的毫秒数）
    console.log('时间戳', e.timeStamp);

    // bubbles：事件是否冒泡（true/false）
    console.log('是否冒泡', e.bubbles);


    // ==================== 目标元素信息 ====================

    // target：触发事件的元素（事件源头）
    // 可能不是绑定事件的元素（例如子元素触发后冒泡到父元素）
    console.log('触发源头', e.target);

    // currentTarget：绑定事件的元素（当前正在处理事件的元素）
    // 这里是实际绑定 bindtap 的元素
    console.log('绑定事件的组件', e.currentTarget);


    // ==================== dataset：元素的自定义数据 ====================

    /*
     * dataset：元素上 data-* 属性对应的对象
     * 例如：data-id="123" data-name="Alice"
     * 在 dataset 中变为：{ id: '123', name: 'Alice' }
     * 注意：data-id 转为 dataset.id（驼峰命名）
     */
    console.log('自定义数据集', e.currentTarget.dataset);

    // 获取具体的值
    const id = e.currentTarget.dataset.id;
    const name = e.currentTarget.dataset.name;


    // ==================== 触摸信息 ====================

    // touches：当前触摸点的列表（数组）
    // 每个触摸点包含：identifier（触摸点ID）、pageX/Y（页面坐标）、clientX/Y（屏幕坐标）
    console.log('触摸点列表', e.touches);

    // changedTouches：变化的触摸点列表
    // 用于 touchend、touchcancel 等事件
    console.log('变化的触摸点', e.changedTouches);


    // ==================== 完整示例 ====================

    // 点击列表项并传递参数
    // WXML: <view bindtap="onItemTap" data-id="{{item.id}}" data-index="{{index}}">
  },

  onItemTap: function(e) {
    // 获取自定义数据属性
    const itemId = e.currentTarget.dataset.id;
    const itemIndex = e.currentTarget.dataset.index;

    // 获取触摸位置
    const touchX = e.touches[0].clientX;
    const touchY = e.touches[0].clientY;

    console.log(`点击了第 ${itemIndex + 1} 项，ID: ${itemId}，坐标: (${touchX}, ${touchY})`);
  }
}
```

### 事件传参

小程序**不支持直接在事件中传参**（如 `bindtap="handleTap(1)"` 是无效的），需要通过自定义属性方式传递参数：

```xml
<!-- ==================== 错误写法 ==================== -->
<!-- 小程序的事件绑定不支持直接传参，以下写法不会生效 -->

<!-- ❌ 错误：直接在事件名后加括号传参 -->
<button bindtap="handleTap(1)">按钮1</button>
<button bindtap="handleTap('hello')">按钮2</button>


<!-- ==================== 正确写法：使用 data-* 属性 ==================== -->
<!-- 通过 data-属性名="值" 的方式定义自定义数据 -->
<!-- 然后在事件处理函数中通过 e.currentTarget.dataset 获取 -->

<!-- 传递单个参数 -->
<button bindtap="handleTap" data-id="1">按钮1</button>

<!-- 传递多个参数 -->
<button bindtap="handleTap" data-id="1" data-type="add">按钮2</button>

<!-- 传递复杂数据（对象需要先 JSON.stringify） -->
<button bindtap="handleTap" data-info="{{JSON.stringify(item)}}">按钮3</button>

<!-- 传递布尔值 -->
<button bindtap="handleTap" data-flag="true">按钮4</button>
<button bindtap="handleTap" data-flag="{{isActive}}">按钮5</button>
```

```javascript
// ==================== 事件处理函数中获取参数 ====================

// 所有参数都通过 e.currentTarget.dataset 获取
handleTap: function(e) {
  // dataset 是一个对象，包含所有 data-* 属性的值
  // data-id="1"       → dataset.id = "1"
  // data-type="add"   → dataset.type = "add"

  // 获取单个参数
  const id = e.currentTarget.dataset.id;
  console.log('ID:', id);  // 输出：1

  // 获取多个参数
  const id = e.currentTarget.dataset.id;
  const type = e.currentTarget.dataset.type;
  console.log('ID和类型:', id, type);  // 输出：1, add

  // 注意：所有 dataset 中的值都是字符串类型

  const numericId = parseInt(e.currentTarget.dataset.id);

  // 获取布尔值
  const flag = e.currentTarget.dataset.flag === 'true';  // 转回布尔值

  // WXML: data-info="{{JSON.stringify(item)}}"
  const info = JSON.parse(e.currentTarget.dataset.info);
}
```

## 自定义组件：复用与解耦

组件化是小程序开发的核心思想之一。通过自定义组件，可以将复杂的页面拆分成独立的、可复用的模块。

### 组件创建与使用

自定义组件是小程序实现代码复用的核心方式。下面详细讲解组件的创建过程：

```bash
# ==================== 组件目录结构 ====================
# 组件通常放在 components 文件夹下，一个组件对应一个文件夹

components/                    # 全局组件目录（可在 app.json 中全局注册）
└── my-button/               # 按钮组件文件夹（建议与组件名一致）
    ├── my-button.js          # 组件逻辑文件（必须）
    ├── my-button.wxml        # 组件结构文件（必须）
    ├── my-button.wxss        # 组件样式文件（必须）
    └── my-button.json        # 组件配置文件（必须）
```

```json
// ==================== components/my-button/my-button.json ====================
{
  // component 字段：声明这是一个自定义组件（必填）
  // 如果缺少此字段，小程序会将其当作普通页面处理
  "component": true,

  // usingComponents：组件内部使用的其他组件
  // 例如：组件内部用到了其他第三方组件
  "usingComponents": {}
}
```

```javascript
// ==================== components/my-button/my-button.js ====================

/*
 * 组件逻辑文件
 * 必须使用 Component() 构造器，不能使用 Page()
 */

Component({

  // ==================== properties：组件属性（对外接口） ====================
  // properties 定义了外部可以向组件传递的数据
  properties: {
    // 按钮文字（外部传入）
    text: {
      type: String,           // 数据类型：必填
      value: '按钮'          // 默认值
    },

    // 是否禁用状态
    disabled: {
      type: Boolean,
      value: false
    },

    // 按钮类型（primary/default/warn）
    type: {
      type: String,
      value: 'default'
    }
  },


  // ==================== data：组件内部状态 ====================
  data: {
    // 组件内部使用的状态，与 properties 的区别：
    // - properties：由外部传入，只读
    // - data：组件内部管理，可修改
    isPressed: false
  },


  // ==================== methods：组件方法 ====================
  methods: {

    // 按钮点击事件处理
    onTap: function() {
      // 如果组件被禁用，不触发点击
      if (this.data.disabled) {
        return;
      }

      // 更新内部状态
      this.setData({ isPressed: true });
      setTimeout(() => {
        this.setData({ isPressed: false });
      }, 100);

      // 向外部触发自定义事件
      // triggerEvent(事件名, 事件数据, 事件选项)
      this.triggerEvent('click', {
        text: this.data.text
      });
    }
  }
});
```

### 组件间通信

组件之间的通信是小程序组件化开发的核心，主要有三种方式：

| 通信方式 | 说明 | 使用场景 |
|---------|------|---------|
| 父 → 子 | 通过属性传递 | 配置组件行为 |
| 子 → 父 | triggerEvent 触发事件 | 通知父组件 |
| 父 → 子 | 获取组件实例调用方法 | 主动调用组件方法 |

```javascript
// ==================== 父组件获取子组件实例 ====================

/*
 * 使用 this.selectComponent('#组件id') 可以获取子组件实例
 * 然后可以调用子组件的公共方法或获取子组件的数据
 */

// 页面 WXML 中给组件添加 id 或 class
// <my-component id="myComponent" class="my-class" />

// 页面 JS 中获取组件实例
const child = this.selectComponent('#myComponent');  // 通过 id 获取
// 或
const child = this.selectComponent('.my-class');    // 通过 class 获取（但只返回第一个匹配项）

// 调用子组件暴露的方法
if (child) {
  // 假设子组件有一个 getData() 方法
  const data = child.getData();
  // 假设子组件有一个 reset() 方法
  child.reset();
}


// ==================== 完整示例：父组件控制子组件 ====================

// pages/index/index.wxml
/*
  <view class="container">
    <!-- 给子组件添加 id，供 selectComponent 使用 -->
    <my-slider id="slider" />

    <!-- 控制按钮 -->
    <button bindtap="onReset">重置滑块</button>
    <button bindtap="onGetValue">获取值</button>
  </view>
*/

Page({
  data: {
    sliderValue: 0
  },

  // 重置滑块：调用子组件方法
  onReset: function() {
    const slider = this.selectComponent('#slider');
    slider.reset(0);  // 调用子组件的 reset 方法，将值重置为 0
  },

  // 获取滑块当前值：调用子组件方法并获取返回值
  onGetValue: function() {
    const slider = this.selectComponent('#slider');
    const value = slider.getValue();  // 获取子组件的当前值
    console.log('当前滑块值:', value);
    this.setData({ sliderValue: value });
  }
});


// components/my-slider/my-slider.js
Component({
  properties: {
    // 滑块的值（外部可设置）
    value: {
      type: Number,
      value: 50
    }
  },

  data: {
    // 内部值（组件自己管理）
    internalValue: 50
  },

  methods: {
    // 获取当前值（供父组件调用）
    getValue: function() {
      return this.data.internalValue;
    },

    // 重置滑块（供父组件调用）
    reset: function(newValue) {
      this.setData({
        internalValue: newValue || 0
      });
    },

    // 内部方法：滑块变化时触发事件通知父组件
    onSliderChange: function(e) {
      const value = e.detail.value;
      this.setData({ internalValue: value });

      // 通过 triggerEvent 通知父组件值已变化
      this.triggerEvent('change', { value: value });
    }
  }
});
```

## 小程序 API：能力的入口

小程序提供了丰富的 API，涵盖网络、存储、设备、媒体等各个方面。

### 网络请求

`wx.request()` 是小程序发送网络请求的核心 API，用于与后端服务器通信：

```javascript
// ==================== wx.request 发起请求 ====================

/*
 * wx.request() 用于发起 HTTPS 请求
 * 完整签名：wx.request(Object object)
 * 返回一个请求任务对象（RequestTask）
 */

wx.request({
  // 必填项：请求地址（需要是已配置的合法域名）
  url: 'https://api.example.com/data',

  // 请求方法，默认是 GET
  // 可选值：GET, POST, PUT, DELETE, HEAD, OPTIONS, TRACE, CONNECT
  method: 'GET',

  // 请求参数（会自动拼接在 URL 后面，GET 请求）或请求体（POST 请求）
  data: {
    page: 1,      // 页码
    size: 10,    // 每页条数
    keyword: '前端'  // 搜索关键词
  },

  // 请求头（可选）
  header: {
    'content-type': 'application/json',  // 默认值，可不设置
    'Authorization': 'Bearer xxx'         // 如果需要登录验证
  },

  // 请求超时时间（单位：毫秒），默认 60000ms（1分钟）
  timeout: 10000,

  // 请求成功的回调函数
  // res：响应对象，包含服务器返回的数据
  success: function(res) {
    // res.statusCode：HTTP 状态码（如 200, 404, 500）
    // res.data：服务器返回的数据（通常是 JSON 对象或数组）
    // res.header：响应头信息

    console.log('状态码:', res.statusCode);
    console.log('返回数据:', res.data);

    // 通常需要判断业务状态码（由后端定义）
    if (res.data.code === 0) {  // 假设 0 表示成功
      const list = res.data.data.list;
      // 处理数据...
    }
  },

  // 请求失败的回调函数
  fail: function(err) {
    // 网络错误、请求超时等情况会触发此回调
    console.error('请求失败:', err);

    // 常见错误：
    // err.errMsg: 错误信息，如 "request:fail url not in domain list"
    wx.showToast({
      title: '网络请求失败',
      icon: 'none'
    });
  },

  // 请求完成后的回调（无论成功或失败都会调用）
  complete: function() {
    // 常用于关闭 loading、隐藏加载状态等
    wx.hideLoading();
  }
});


// ==================== 请求任务对象 ====================

/*
 * wx.request() 返回一个请求任务对象
 * 通过这个对象可以中断请求或获取请求进度
 */

const requestTask = wx.request({
  url: 'https://api.example.com/download',
  success: function(res) {
    console.log(res.data);
  }
});

// 中断请求
// requestTask.abort();


// ==================== 回调函数写法对比 ====================

// 写法1：完整回调函数（上述示例）
wx.request({
  url: '...',
  success: function(res) {},
  fail: function(err) {},
  complete: function() {}
});

// 写法2：Promise 封装（推荐，实际开发中使用）
const request = (options) => {
  return new Promise((resolve, reject) => {
    wx.request({
      ...options,
      success: resolve,
      fail: reject,
      complete: resolve  // 或单独处理
    });
  });
};

// 写法3：箭头函数简化
wx.request({
  url: '...',
  success: (res) => { /* ... */ },
  fail: (err) => { /* ... */ }
});
```

:::warning
上线前必须在微信公众平台配置**合法域名**，否则请求会失败。本地开发时可在微信开发者工具中勾选「不校验合法域名」。
:::

### Promise 化封装

原生 API 采用回调写法，实际开发中建议封装为 Promise 风格以便于使用 async/await：

```javascript
// ==================== utils/request.js - 请求封装示例 ====================

/*
 * 将 wx.request 封装为 Promise 风格
 * 封装后的代码可以使用 async/await 语法，更加简洁易读
 */

// 基础封装
const request = (options) => {
  return new Promise((resolve, reject) => {
    wx.request({
      ...options,
      // 成功时调用 resolve
      success: (res) => {
        // 可以在这里统一处理业务状态码
        if (res.data.code === 0) {
          resolve(res.data);
        } else {
          // 业务错误，如登录失效、权限不足等
          wx.showToast({
            title: res.data.message || '请求失败',
            icon: 'none'
          });
          reject(res.data);
        }
      },
      // 失败时调用 reject
      fail: (err) => {
        wx.showToast({
          title: '网络请求失败',
          icon: 'none'
        });
        reject(err);
      }
    });
  });
};


// ==================== 使用 async/await 调用 ====================

// 在 Page 或 Component 的 methods 中使用
methods: {

  // 获取文章列表
  async fetchArticleList() {
    // 显示加载状态
    wx.showLoading({ title: '加载中...' });

    try {
      // 使用 await 等待请求完成（代码会在这里暂停，直到请求返回）
      const res = await request({
        url: 'https://api.example.com/articles',
        method: 'GET',
        data: { page: 1, size: 10 }
      });

      // 请求成功，更新页面数据
      this.setData({
        list: res.data.list,
        total: res.data.total
      });

    } catch (err) {
      // 请求失败，错误已经在封装中处理过
      console.error('获取文章列表失败', err);
    } finally {
      // 无论成功或失败，都会执行这里
      wx.hideLoading();
    }
  },


  // 提交表单
  async submitForm(formData) {
    wx.showLoading({ title: '提交中...' });

    try {
      const res = await request({
        url: 'https://api.example.com/submit',
        method: 'POST',
        data: formData
      });

      wx.showToast({
        title: '提交成功',
        icon: 'success'
      });

      // 提交成功后跳转
      wx.navigateBack();

    } catch (err) {
      // 错误处理
      console.error('提交失败', err);
    } finally {
      wx.hideLoading();
    }
  }
}


// ==================== 其他常用 API 的 Promise 封装 ====================

// 微信登录
const login = () => {
  return new Promise((resolve, reject) => {
    wx.login({
      success: resolve,
      fail: reject
    });
  });
};

// 获取用户信息
const getUserInfo = () => {
  return new Promise((resolve, reject) => {
    wx.getUserProfile({
      desc: '用于完善会员资料',
      success: resolve,

    });
  });
};

// 选择图片
const chooseImage = (count = 1) => {
  return new Promise((resolve, reject) => {
    wx.chooseImage({
      count: count,          // 选择图片的数量
      sizeType: ['compressed'],  // 是否压缩
      sourceType: ['album', 'camera'],  //图片来源
      success: resolve,
      fail: reject
    });
  });
};

// 使用示例
async onSelectAvatar() {
  try {
    const res = await chooseImage(1);
    const avatarUrl = res.tempFilePaths[0];
    console.log('选择的图片路径:', avatarUrl);
  } catch (err) {
    console.error('选择图片失败', err);
  }
}
```

### 存储 API

小程序提供了本地数据存储能力，用于持久化保存用户数据：

| API | 说明 | 同步版本 |
|-----|------|---------|
| wx.setStorage | 异步存储数据 | wx.setStorageSync |
| wx.getStorage | 异步获取数据 | wx.getStorageSync |
| wx.removeStorage | 异步删除数据 | wx.removeStorageSync |
| wx.clearStorage | 异步清空所有数据 | wx.clearStorageSync |

```javascript
// ==================== 异步版本 API ====================

// 存储数据到本地
// key：存储的键名（字符串）
// data：要存储的数据（可以是任意类型，会被 JSON.stringify 处理）
wx.setStorage({
  key: 'userInfo',              // 键名
  data: {                       // 要存储的数据
    name: '小明',
    age: 18,
    avatar: 'https://example.com/avatar.jpg'
  },
  success: function() {
    console.log('存储成功');
  },
  fail: function(err) {
    console.error('存储失败', err);
  }
});

// 从本地存储中获取数据
wx.getStorage({
  key: 'userInfo',              // 要获取的键名
  success: function(res) {
    // res.data：获取到的数据（如果是对象，会被 JSON.parse 处理）
    console.log('获取到的数据:', res.data);
    const userInfo = res.data;
  },
  fail: function(err) {
    console.error('获取失败，键名可能不存在', err);
  }
});

// 删除指定键名的数据
wx.removeStorage({
  key: 'userInfo',
  success: function() {
    console.log('删除成功');
  }
});

// 清空全部本地存储数据
wx.clearStorage({
  success: function() {
    console.log('清空成功');
  }
});


// ==================== 同步版本 API（与上述异步版本功能相同） ====================

/*
 * 同步版本 API 以 Sync 结尾
 * 同步版本会阻塞代码执行，直接返回结果
 * 适用于简单的数据存取，但可能影响性能
 */

// 存储数据
wx.setStorageSync('token', 'abc123');
wx.setStorageSync('settings', { theme: 'dark', language: 'zh-CN' });

// 获取数据（如果键名不存在，返回空字符串''）
const token = wx.getStorageSync('token');
const settings = wx.getStorageSync('settings');

// 删除数据
wx.removeStorageSync('token');

// 清空全部
wx.clearStorageSync();


// ==================== 存储 API 的注意事项 ====================

/*
 * 1. 存储限制：
 *    - 单个 key 允许存储的最大数据量为 1MB
 *    - 总存储容量为 10MB
 *    - 超出限制会导致存储失败
 *
 * 2. 数据类型：
 *    - 支持任何可以 JSON.stringify 的数据类型
 *    - 数字、字符串、布尔值、数组、对象等
 *
 * 3. 同步 vs 异步：
 *    - 同步 API 会阻塞代码执行
 *    - 建议在页面初始化时使用同步读取
 *    - 用户操作触发的存储使用异步
 */

// ==================== 实际使用示例 ====================

Page({
  data: {
    userInfo: null
  },

  onLoad: function() {
    // 页面加载时同步读取（阻塞式，但数据量小影响不大）
    const userInfo = wx.getStorageSync('userInfo');
    if (userInfo) {
      this.setData({ userInfo: userInfo });
    }
  },

  // 登录成功后存储用户信息
  onLoginSuccess: function(userInfo) {
    // 异步存储，不阻塞页面响应
    wx.setStorage({
      key: 'userInfo',
      data: userInfo,
      success: () => {
        this.setData({ userInfo: userInfo });
      }
    });
  },

  // 退出登录时清除用户信息
  onLogout: function() {
    wx.removeStorage({
```

## 实战技巧：踩坑总结

### setData 性能优化

`setData` 是小程序中最容易引发性能问题的操作。以下是经验总结：

```javascript
// ==================== setData 的工作原理 ====================

/*
 * setData 的执行过程：
 * 1. 将数据从逻辑层发送到视图层（跨线程通信）
 * 2. 视图层接收到数据后进行页面渲染
 *
 * 性能问题来源：
 * - 每次 setData 都会触发跨线程通信
 * - 数据量越大，通信耗时越长
 * - 频繁调用 setData 会导致卡顿
 */


// ==================== 低效写法1：每次合并新数据都创建新数组 ====================

// ❌ 低效：在 setData 中直接 concat，每次都创建新数组
// 问题：每次 setData 都会传输整个新数组
this.setData({
  list: this.data.list.concat(newItem)  // concat 会创建新数组
});

// ✅ 高效：先修改原数组，再整体更新
const list = this.data.list;    // 获取原数组引用
list.push(newItem);              // 修改原数组
this.setData({
  list: list                     // 传数组引用，而非新数组
});


// ==================== 低效写法2：频繁调用 setData ====================

// ❌ 低效：在循环中多次调用 setData
// 问题：每次迭代都会触发一次跨线程通信
for (let i = 0; i < 100; i++) {
  this.setData({ count: i });  // 100 次通信！
}

// ✅ 高效：合并多次更新为一次
for (let i = 0; i < 100; i++) {
  // 只更新数据，不触发 setData
}
this.setData({ count: 99 });    // 最后一次性更新


// ==================== 低效写法3：传输与渲染无关的数据 ====================

// ❌ 低效：将大量与视图无关的数据放在 data 中
Page({
  data: {
    // 这些数据不需要渲染，但每次 setData 都会传输
    largeDataArray: hugeArray,   // 很大的数组
    tempCalculation: null,       // 临时计算结果
    someCacheData: {}           // 缓存数据
  }
});

// ✅ 高效：将不渲染的数据放在 data 外部
Page({
  // 只放需要渲染的数据
  data: {
    displayList: []
  },

  // 不渲染的数据放在 this 上
  someCacheData: {},
  tempCalculation: null
});


// ==================== 低效写法4：频繁更新对象中的某个属性 ====================

// ❌ 低效：逐个更新对象属性
this.setData({ 'user.name': '小明' });
this.setData({ 'user.age': 18 });
this.setData({ 'user.city': '北京' });

// ✅ 高效：一次性更新整个对象
this.setData({
  user: {
    name: '小明',
    age: 18,
    city: '北京'
  }
});


// ==================== 高效 setData 的最佳实践 ====================

// 1. 使用局部变量缓存数据，减少 setData 次数
onScroll: function(e) {
  // 缓存滚动值，但不立即 setData
  this._scrollTop = e.scrollTop;

  // 节流：每 100ms 更新一次
  if (!this._scrollTimer) {
    this._scrollTimer = setTimeout(() => {
      this.setData({ scrollTop: this._scrollTop });
      this._scrollTimer = null;
    }, 100);
  }
},

// 2. 对于列表数据，使用索引更新而非整体更新
updateItem: function(index, newItem) {
  // ❌ 低效：整体更新
  // this.setData({ list: this.data.list });

  // ✅ 高效：只更新变化的项
  const key = 'list[' + index + ']';
  this.setData({
    [key]: newItem
  });
},

// 3. 页面隐藏时暂停更新
onHide: function() {
  this._paused = true;
},
onShow: function() {
  this._paused = false;
  if (this._pendingData) {
    this.setData(this._pendingData);
    this._pendingData = null;
  }
}
```

```javascript
// ==================== 方式1：styleIsolation 配置 ====================
```

### 定时器清理

页面卸载时必须清理定时器，否则会造成内存泄漏：

```javascript
Page({

  data: {
    // 定时器 ID
    timer: null,
    // 多个定时器可以用数组存储
    timers: []
  },


  // ==================== 页面加载时启动定时器 ====================
  onLoad: function() {

    // 启动单个定时器
    this.data.timer = setInterval(() => {
      console.log('tick');

      // 执行定时任务...
      this.fetchUpdate();
    }, 3000);  // 每 3 秒执行一次


    // 启动多个定时器（如果有需要）
    this.data.timers.push(
      setTimeout(() => { this.autoSave(); }, 5000)
    );
    this.data.timers.push(
      setInterval(() => { this.checkNetwork(); }, 10000)
    );
  },


  // ==================== 页面卸载时清理定时器 ====================
  onUnload: function() {

    // 清理单个定时器
    if (this.data.timer) {
      clearInterval(this.data.timer);
      this.data.timer = null;
    }


    // 清理多个定时器
    this.data.timers.forEach(timerId => {
      clearInterval(timerId);
      clearTimeout(timerId);
    });
    this.data.timers = [];
  },


  // ==================== 组件中的定时器清理 ====================
  // 如果在组件中使用定时器，同样需要在 detached 生命周期中清理
  lifetimes: {
    attached: function() {
      this.data.animationTimer = setInterval(() => {
        this.updateAnimation();
      }, 16);  // 约 60fps 的动画
    },
    detached: function() {
      if (this.data.animationTimer) {
        clearInterval(this.data.animationTimer);
        this.data.animationTimer = null;
      }
    }
  }
});
```
