## 引言：站在终点回望

2025年6月22日，最后一天。

上午九点，我像往常一样刷卡进入公司。但今天不是来「上班」的——是来完成工作交接，是来和前辈们道别。

整理工位的时候，我翻出了这18周来的所有开发日志。从第一周的「入职第一天，什么都不懂」到第十六周的「版本更新说明编写完成」，每一页都是真实的成长轨迹。

18周前，我躺在宿舍床上忐忑不安；18周后，我在公司的最后一天写下这段话：

**「这趟旅程，超出了我的预期。」**

不是在技术上——虽然确实学到了很多，而是在认知上。我对「企业开发」「团队协作」「工程师」这些词的理解，都因为这段经历而被重塑了。

这篇文章，是这段18周旅程的最后一篇，也是整个实习系列的收尾。

## 企业教给我的那些事

### 职业素养：第一课

入职第一天，mentor没给我安排任务，只是说了一句：「明天九点准时到就行。」

我当时觉得这是客套话。后来才发现，**准时是企业最基本的信用**。在学校里，迟到几分钟找借口还能过关；在公司里，每一次迟到都在消耗别人对你的信任。

这种对「守时」的要求，只是职业素养的冰山一角。代码要按时提交、会议要准时参加、承诺的进度不能打折扣——这些看似简单的事，在企业里有着完全不同的分量。

### 「可维护」比「能跑」更重要

学校里，代码能跑就算交差了。但mentor第一次review我的代码时，没有问我「功能实现了吗」，而是问「这段代码三个月后你还能看懂吗」。

这个问题让我愣了很久。

在企业里，代码不是写完就结束了。后续可能有人接手、可能有人扩展、可能有人debug。**代码的可读性、可维护性，比炫技重要得多。** 注释要写清楚为什么要这样做，而不是做了什么；变量命名要让人一眼看懂含义，而不是用a、b、c糊弄。

mentor有句话很直接：「写代码的时候，假设下一个看这段代码的人是个暴躁的老手，他只想快点解决问题，不想猜你在想什么。」

### 有人愿意带你，是幸运

在学校里，老师教你是因为职责；在公司里，前辈愿意花时间带你，是额外的付出。

mentor经常在完成自己的工作之外，还要回答我的各种「小白问题」。有时候问多了我自己都不好意思，但他总说：「问清楚总比做错了强。」

有一次我问他：「您花这么多时间带我，不耽误自己的事吗？」

他说：「当年我也是被带过来的。等你以后当了mentor，就明白了。」

这句话让我第一次理解了什么叫「传承」。

### 细节规范，是效率的保障

刚进公司时，我觉得那些「代码格式规范」「命名规范」「日志规范」都是形式主义——代码能跑不就行了？

后来我才理解，规范的本质是**降低协作成本**。当所有人都遵循同一套规范，代码审查可以专注于逻辑而不是格式；当天志记录格式统一，排查问题可以快速定位而不是大海捞针。

### 交付意识：从学习到工作

在学校里，我的目标是「学会」；在公司里，我的目标是「交付」。

这两个目标的差异，体现在方方面面：

学习可以慢慢来，不懂的地方可以多花时间研究；工作有截止日期，必须在规定时间内完成。学习可以追求完美，理解透彻了再做；工作要先交付能用版本，再迭代优化。

mentor有句话：「完成优于完美。完美是动态的，但交付是此时此刻的。」

这句话帮我完成了从学生到职场人的关键转变。

### 实习教会我的六件事

**第一：清楚自己的边界。** 实习生不能碰核心业务逻辑，这是原则。知道自己能做什么、不能做什么，比盲目逞强更重要。

**第二：主动汇报，别等别人来问。** 在学校里，等老师问你是常态；在公司里，mentor没空盯着你。主动同步进度、主动反馈问题，这才是靠谱的表现。

**第三：工时记录是认真的。** 在Jira上填工时，不是在应付任务，而是对自己时间的尊重、对团队进度的负责。

**第四：交接比完成更重要。** 离开时把工作交代清楚，文档归档、代码注释、遗留问题——这些决定了别人接手的难度，也体现了你的职业素养。

**第五：环境稳定是责任。** 测试环境出问题不是小事，会影响整个团队的进度。主动监控、及时反馈，这些小事体现了大局观。

**第六：mentor的话要记牢。** 「问清楚总比做错了强」「完成优于完美」「修坏了可以回滚，但不敢迈出去就永远长不大」——这些话，值得刻在心里。

## 实习与实训：两次不同的「企业初体验」

| 维度 | 工匠工坊            | 浙江索思实习           |
| -- | --------------- | ---------------- |
| 性质 | 学校组织的训练，在校园内完成  | 真正的公司，在写字楼里上班    |
| 时长 | 13周             | 18周              |
| 角色 | 学生，以学习为主        | 实习生，以配合为主        |
| 项目 | 练手项目，不涉及真实业务    | 真实项目，可能影响实际使用    |
| 指导 | 企业导师进课堂，课后各自散去  | mentor每天在身边，随时可问 |
| 压力 | 较小，deadline比较弹性 | 真实，任务有明确截止日期     |
| 收获 | 体验了团队协作的规范      | 理解了职场规则和职业素养     |

工匠工坊让我知道了「企业开发是什么样」；浙江索思让我体会了「在企业里工作是什么感觉」。

前者是「模拟」，后者是「真实」。

没有工匠工坊的打基础，我可能不会那么快适应实习的节奏；没有实习的真实体验，我也无法真正理解工匠工坊导师说的那些话的含义。

两段经历，缺一不可。

## 结语：真正的「第一站」

2025年6月22日，浙江索思科技有限公司JAVA实习正式结束。

拿到实习评价表的时候，我仔细看了一遍 mentor 写的评语。其中有一句话让我印象深刻：

> 「在实习期间，始终保持谦逊好学的态度，能够严格按照企业工作流程完成各项辅助任务，展现出了良好的学习潜力和团队协作能力。」

说实话，看到「谦逊好学」这四个字，我有种被看透了的感觉。

这四个月，我确实还有很多不足：

- 技术深度有限，核心业务逻辑还是依赖前辈
- 问题排查效率有待提升，有时候会卡在简单问题上
- 全局视野不够强，有时候只盯着自己那一块

但我也有实实在在的收获：

- 知道了企业开发的规范是什么
- 体验了真实职场的节奏和规则
- 理解了「配合」和「交付」的区别
- 建立了从学生到职场人的初步心态

mentor在最后一天说：「这四个月你学到的东西，可能只是冰山一角。但你已经有了最重要的东西——正确的态度和方法。」

这句话我会一直记着。

下一段旅程，或许是更深入的技术学习，或许是真正的全职工作。不管是什么，带着这18周的积累，我不会空手而去。
