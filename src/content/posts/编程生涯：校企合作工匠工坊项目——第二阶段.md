## 项目选型：为什么是「每日打卡」

### 为什么选这个项目

企业导师在第二阶段开始时给出了几个备选项目方向，让我们自行选择。经过考虑，我选了「每日打卡」小程序，理由有三：

1. **功能边界清晰**——打卡、记录、统计，三大核心功能，不会过于复杂
2. **能串联第一阶段知识**——列表渲染、表单交互、本地存储，都要用到
3. **真实场景需求**——我自己确实需要一个记账类的小工具来培养每日习惯

### 项目功能规划

「每日打卡」的核心功能模块：

| 模块       | 功能描述            | 对应技术点          |
| -------- | --------------- | -------------- |
| **打卡首页** | 展示今日打卡状态、一键打卡按钮 | 条件渲染、数据绑定、事件处理 |
| **打卡记录** | 按日期查看历史打卡记录     | 列表渲染、日期筛选      |
| **数据统计** | 展示打卡天数、连续打卡天数   | 数据计算、图表展示      |
| **习惯管理** | 添加、编辑、删除打卡习惯    | 表单处理、本地存储      |

:::note
项目采用「本地存储 + 模拟接口」的方式实现数据持久化，暂不涉及真实后端联调。接口联调的内容留到第三阶段小组协作时再实践。
:::

## 核心功能实战

### 打卡首页：看似简单，暗藏玄机

#### 布局结构

打卡首页的布局看起来很简单——一个状态展示区、一个打卡按钮、一个打卡记录列表。第一阶段学的flex布局和rpx单位完全够用：

```vue
<!-- pages/index/index.vue -->
<template>
  <view class="checkin-page">
    <!-- 今日状态卡片 -->
    <view class="today-card">
      <view class="date-info">
        <text class="weekday">{{ weekdayText }}</text>
        <text class="date">{{ todayDate }}</text>
      </view>
      <view class="status-badge" :class="{ checked: todayChecked }">
        {{ todayChecked ? '今日已打卡' : '今日待打卡' }}
      </view>
    </view>

    <!-- 一键打卡按钮 -->
    <button
      class="checkin-btn"
      :class="{ active: !todayChecked }"
      @tap="handleCheckin"
    >
      {{ todayChecked ? '已打卡 ✓' : '立即打卡' }}
    </button>

    <!-- 今日打卡习惯列表 -->
    <view class="habit-list">
      <view class="section-title">今日习惯</view>
      <view
        v-for="habit in todayHabits"
        :key="habit.id"
        class="habit-item"
        @tap="toggleHabit(habit.id)"
      >
        <view class="habit-checkbox" :class="{ checked: habit.done }">
          <text v-if="habit.done" class="check-icon">✓</text>
        </view>
        <text class="habit-name">{{ habit.name }}</text>
        <text class="habit-time">{{ habit.reminderTime }}</text>
      </view>
    </view>
  </view>
</template>
```

#### 数据结构设计

```javascript
<script>
export default {
  data() {
    return {
      todayDate: '',  // 今日日期字符串
      weekdayText: '',  // 星期几
      todayChecked: false,  // 今日是否已打卡
      todayHabits: []  // 今日习惯列表
    }
  },
  onShow() {
    this.loadTodayData()
  },
  methods: {
    loadTodayData() {
      const now = new Date()
      this.todayDate = `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}-${String(now.getDate()).padStart(2, '0')}`
      this.weekdayText = ['星期日', '星期一', '星期二', '星期三', '星期四', '星期五', '星期六'][now.getDay()]

      // 从本地存储加载数据
      const storageData = uni.getStorageSync('checkin_records')
      const records = storageData ? JSON.parse(storageData) : {}

      // 检查今日打卡状态
      if (records[this.todayDate]) {
        this.todayChecked = true
        this.todayHabits = records[this.todayDate].habits
      } else {
        this.todayChecked = false
        this.todayHabits = uni.getStorageSync('habit_list') || []
      }
    },
    handleCheckin() {
      if (this.todayChecked) return

      const now = new Date()
      const dateKey = this.todayDate
      let records = uni.getStorageSync('checkin_records')
      records = records ? JSON.parse(records) : {}

      // 标记所有习惯为已完成
      this.todayHabits.forEach(habit => {
        habit.done = true
        habit.checkinTime = `${String(now.getHours()).padStart(2, '0')}:${String(now.getMinutes()).padStart(2, '0')}`
      })

      // 保存打卡记录
      records[dateKey] = {
        habits: this.todayHabits,
        checkinTime: now.toISOString()
      }

      uni.setStorageSync('checkin_records', JSON.stringify(records))
      this.todayChecked = true

      uni.showToast({
        title: '打卡成功！',
        icon: 'success',
        duration: 1500
      })
    },
    toggleHabit(habitId) {
      const habit = this.todayHabits.find(h => h.id === habitId)
      if (habit) {
        habit.done = !habit.done
      }
    }
  }
}
</script>
```

:::note
这里的 `uni.getStorageSync` 和 `uni.setStorageSync` 是uniapp提供的**本地存储API**。第一阶段的demo里没有涉及这个知识点，属于「用到才学」的新内容。这也印证了一个道理：**项目的需求会倒逼你学习新的知识**。
:::

### 打卡记录：列表渲染的进阶

打卡记录页面需要展示历史打卡数据，这里涉及到**数据的筛选和格式化**：

```vue
<!-- pages/record/record.vue -->
<template>
  <view class="record-page">
    <!-- 月份选择器 -->
    <view class="month-selector">
      <view class="month-btn" @tap="prevMonth">
        <text>◀</text>
      </view>
      <text class="current-month">{{ currentYear }}年{{ currentMonth }}月</text>
      <view class="month-btn" @tap="nextMonth">
        <text>▶</text>
      </view>
    </view>

    <!-- 打卡日历视图 -->
    <view class="calendar-grid">
      <view
        v-for="day in calendarDays"
        :key="day.date"
        class="calendar-day"
        :class="{
          'has-record': day.hasRecord,
          'full-complete': day.fullComplete,
          'partial': day.partialComplete,
          'empty': !day.hasRecord,
          'future': day.isFuture
        }"
      >
        <text class="day-number">{{ day.day }}</text>
        <view v-if="day.hasRecord" class="record-indicator"></view>
      </view>
    </view>

    <!-- 月度统计 -->
    <view class="month-stats">
      <view class="stat-item">
        <text class="stat-value">{{ monthStats.totalDays }}</text>
        <text class="stat-label">打卡天数</text>
      </view>
      <view class="stat-item">
        <text class="stat-value">{{ monthStats.continueDays }}</text>
        <text class="stat-label">连续天数</text>
      </view>
      <view class="stat-item">
        <text class="stat-value">{{ monthStats.completionRate }}%</text>
        <text class="stat-label">完成率</text>
      </view>
    </view>

    <!-- 历史记录列表 -->
    <view class="record-list">
      <view
        v-for="record in filteredRecords"
        :key="record.date"
        class="record-card"
      >
        <view class="record-date">
          <text class="date-text">{{ formatDate(record.date) }}</text>
          <text class="weekday-text">{{ getWeekday(record.date) }}</text>
        </view>
        <view class="record-detail">
          <text class="habit-count">已完成 {{ record.doneCount }} / {{ record.totalCount }} 项</text>
        </view>
      </view>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      currentYear: new Date().getFullYear(),
      currentMonth: new Date().getMonth() + 1,
      calendarDays: [],
      monthStats: {
        totalDays: 0,
        continueDays: 0,
        completionRate: 0
      }
    }
  },
  computed: {
    filteredRecords() {
      const records = uni.getStorageSync('checkin_records') || {}
      const prefix = `${this.currentYear}-${String(this.currentMonth).padStart(2, '0')}`
      return Object.keys(records)
        .filter(date => date.startsWith(prefix))
        .map(date => {
          const habits = records[date].habits
          return {
            date,
            habits,
            doneCount: habits.filter(h => h.done).length,
            totalCount: habits.length
          }
        })
        .sort((a, b) => b.date.localeCompare(a.date))
    }
  },
  onShow() {
    this.loadCalendarData()
  },
  methods: {
    loadCalendarData() {
      // 生成日历数据
      const daysInMonth = new Date(this.currentYear, this.currentMonth, 0).getDate()
      const firstDayWeekday = new Date(this.currentYear, this.currentMonth - 1, 1).getDay()

      const records = uni.getStorageSync('checkin_records') || {}
      this.calendarDays = []

      // 填充空白格子（月初前的空格）
      for (let i = 0; i < firstDayWeekday; i++) {
        this.calendarDays.push({ day: '', empty: true })
      }

      // 生成日历格子
      for (let day = 1; day <= daysInMonth; day++) {
        const dateStr = `${this.currentYear}-${String(this.currentMonth).padStart(2, '0')}-${String(day).padStart(2, '0')}`
        const isFuture = new Date(dateStr) > new Date()
        const record = records[dateStr]

        let hasRecord = false
        let fullComplete = false
        let partialComplete = false

        if (record) {
          hasRecord = true
          const habits = record.habits || []
          const doneCount = habits.filter(h => h.done).length
          fullComplete = doneCount === habits.length && habits.length > 0
          partialComplete = doneCount > 0 && doneCount < habits.length
        }

        this.calendarDays.push({
          day,
          date: dateStr,
          hasRecord,
          fullComplete,
          partialComplete,
          isFuture
        })
      }

      this.calculateStats()
    },
    calculateStats() {
      const records = uni.getStorageSync('checkin_records') || {}
      let totalDays = 0
      let continueDays = 0

      // 简化统计逻辑
      Object.keys(records).forEach(date => {
        if (date.startsWith(`${this.currentYear}-${String(this.currentMonth).padStart(2, '0')}`)) {
          totalDays++
        }
      })

      this.monthStats.totalDays = totalDays
      this.monthStats.continueDays = continueDays
      this.monthStats.completionRate = totalDays > 0 ? Math.round((totalDays / 31) * 100) : 0
    },
    formatDate(dateStr) {
      const parts = dateStr.split('-')
      return `${parseInt(parts[1])}月${parseInt(parts[2])}日`
    },
    getWeekday(dateStr) {
      const weekday = new Date(dateStr).getDay()
      return ['周日', '周一', '周二', '周三', '周四', '周五', '周六'][weekday]
    },
    prevMonth() {
      if (this.currentMonth === 1) {
        this.currentMonth = 12
        this.currentYear--
      } else {
        this.currentMonth--
      }
      this.loadCalendarData()
    },
    nextMonth() {
      if (this.currentMonth === 12) {
        this.currentMonth = 1
        this.currentYear++
      } else {
        this.currentMonth++
      }
      this.loadCalendarData()
    }
  }
}
</script>
```

:::note
这里用到了 `computed` 计算属性，它是Vue.js的核心特性之一。不同于 `data` 中的普通数据，计算属性会根据其依赖的数据自动缓存计算结果。第一阶段只接触了 `methods`，第二阶段才真正用上 `computed`——这也是「项目需求驱动学习」的体现。
:::

## 踩坑实录：理论与实践的Gap

### 问题一：本地存储的「同步」与「异步」

**问题现象**：
打卡成功后，数据明明已经保存了，但退出页面再进来时，有时能显示数据，有时不能。

**根因分析**：
uniapp的存储API有两套：`uni.getStorageSync` / `uni.setStorageSync`（同步）和 `uni.getStorage` / `uni.setStorage`（异步）。我在保存时用了同步API，但加载时有时用同步、有时用异步，导致时序问题。

```javascript
// 错误写法：混用同步和异步
setStorageSync('checkin_records', JSON.stringify(records))  // 同步保存
getStorage({ success: res => { ... } })  // 异步读取，可能在保存完成前执行
```

**解决方案**：
统一使用同步API，或者统一使用异步API。代码中统一使用同步版本，避免时序问题。

```javascript
// 正确写法：统一使用同步API
const records = uni.getStorageSync('checkin_records')
const parsed = records ? JSON.parse(records) : {}
```

**经验总结**：

- 同步API：`Sync`后缀，代码直观，但会阻塞主线程
- 异步API：无后缀，需要回调或Promise处理
- 建议在简单场景下使用同步API，复杂场景使用异步

### 问题二：日期对象的时间陷阱

**问题现象**：
打卡记录显示的日期比实际早一天。比如今天12月3日打卡，记录里显示的是12月2日。

**根因分析**：
JavaScript的 `Date` 对象在格式化时存在**时区偏移**问题。当使用 `toISOString()` 或直接拼接日期字符串时，浏览器会按照UTC时间处理，导致8小时时差（北京时间UTC+8）。

```javascript
// 错误写法
const now = new Date()
const dateStr = now.toISOString().split('T')[0]  // 会出现时差问题
```

**解决方案**：
使用本地时间方法手动拼接，避免时区转换：

```javascript
// 正确写法
const now = new Date()
const dateStr = `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}-${String(now.getDate()).padStart(2, '0')}`
```

**经验总结**：

- 处理日期时，优先使用本地时间方法（getFullYear、getMonth、getDate）
- 避免直接使用 toISOString() 处理日期字符串
- 可以封装一个日期格式化工具函数，统一处理

### 问题三：页面间数据传递

**问题现象**：
从打卡首页跳转到习惯管理页面，添加新习惯后返回，打卡首页的数据没有更新。

**根因分析**：
uniapp的页面跳转不会自动共享数据状态。每个页面都是独立的实例，`onShow` 生命周期只在页面首次加载时触发，返回时不会重新执行。

**解决方案**：

1. 利用 `onShow` 生命周期——每次页面显示时都会执行
2. 在返回时使用 `uni.navigateBack()`，它会触发前一个页面的 `onShow`

```javascript
// 习惯管理页面添加习惯后
addHabit(newHabit) {
  const habits = uni.getStorageSync('habit_list') || []
  habits.push(newHabit)
  uni.setStorageSync('habit_list', habits)

  // 返回上一页（会自动触发首页的 onShow）
  uni.navigateBack()
}
```

**经验总结**：

- 页面数据更新优先考虑 `onShow` 而非 `onLoad`
- 复杂数据共享可以考虑 Vuex（但第二阶段暂未涉及）

## 调试方法：从「盲猜」到「排查」

第二阶段最大的收获，不仅仅是项目本身，更是调试能力的提升。

### 断点调试：HBuilderX的「时光机」

HBuilderX支持在代码中设置断点，当程序执行到断点时会暂停，可以查看当前变量的值。这对于定位问题非常有效。

**操作步骤**：

1. 在代码行号左侧单击，设置断点
2. 点击「运行」→「运行到调试器」
3. 程序会在断点处暂停
4. 使用「调试视图」查看变量值
5. 按 F8 继续执行，或 F10 单步执行

```javascript
// 设置断点，排查数据保存问题
handleCheckin() {
  const now = new Date()  // 断点1：查看当前时间
  const dateKey = this.todayDate  // 断点2：查看日期字符串

  let records = uni.getStorageSync('checkin_records')  // 断点3：查看存储状态
  records = records ? JSON.parse(records) : {}

  this.todayHabits.forEach(habit => {
    habit.done = true  // 断点4：查看循环中的状态变化
  })

  records[dateKey] = { habits: this.todayHabits, checkinTime: now.toISOString() }
  uni.setStorageSync('checkin_records', JSON.stringify(records))  // 断点5：查看最终保存数据
}
```

### console.log：老派但有效

断点调试虽然强大，但有时候 `console.log` 更快捷——尤其是想快速查看某个变量在特定时刻的值时。

```javascript
// 在关键位置打印日志
loadTodayData() {
  console.log('=== loadTodayData 开始 ===')
  console.log('todayDate:', this.todayDate)

  const storageData = uni.getStorageSync('checkin_records')
  console.log('原始存储数据:', storageData)

  const records = storageData ? JSON.parse(storageData) : {}
  console.log('解析后数据:', records)

  if (records[this.todayDate]) {
    console.log('找到今日打卡记录')
    this.todayChecked = true
    this.todayHabits = records[this.todayDate].habits
  }
  console.log('最终 todayHabits:', this.todayHabits)
}
```

:::note
企业导师的一个建议：**给console.log加上前缀标记**，比如 `=== loadTodayData ===` 或 `[DEBUG]`，这样在控制台日志较多时更容易定位是哪个函数的输出。
:::

### 「排除法」：定位问题的利器

当问题原因不明确时，排除法是最实用的策略：

1. **确认数据源**：数据是从本地存储读取的、从接口获取的，还是页面间传递的？
2. **确认数据格式**：JSON是否正确解析？字段名是否拼写一致？
3. **确认执行时序**：代码是否按预期顺序执行？异步操作是否等待了？
4. **确认平台差异**：某些API在不同端（微信/支付宝/H5）表现是否一致？
