## 引言：站在终点回望

2025年1月12日，农历腊月十三。

工匠工坊的最后一堂课，企业导师没有讲新技术。他站在讲台上，问了一个看似简单的问题：

> 「这13周下来，你们觉得自己最大的收获是什么？」

教室里安静了几秒。有人说是Git，有人说是Vue，有人说是「终于知道团队开发是怎么回事了」。

而我的答案是：**知道自己还差得远。**

这不是谦虚。从第一阶段的「会用」，到第二阶段的「能跑」，再到第三阶段的「能协作」，每跨过一道坎，都发现前面还有更高的山。但正因为有了这段经历，我知道这些山是可以翻的。

## 13周的旅程地图

### 时间线与阶段交付

| 阶段   | 时间     | 主题   | 核心交付                      |
| ---- | ------ | ---- | ------------------------- |
| 第一阶段 | 第1-4周  | 框架入门 | 掌握uniapp基础语法，独立完成任务卡片demo |

第三阶段各小组进行了成果展示。我们小组交付的「习惯打卡」小程序，最终获得了**第二名**（共六个小组）。

:::note
名次不是最重要的。更重要的是，在13周的时间里，我们真正体验了「从需求分析到产品交付」的全流程。

## 技术认知的升级

### 从「会皮毛」到「懂门道」

13周下来，uniapp不再是陌生的名词。更重要的是，我建立了一套学习新框架的方法论：

**看文档 → 写demo → 理解概念 → 项目验证**

这个闭环让我面对任何新框架时都不再发怵。

## 协作能力的突破

### Git：协作的基石

没有Git之前，多人同时改代码简直是噩梦——你覆盖我、我覆盖你，谁都说不清最终版本。

有了Git不代表就万事大吉。真正的挑战是**规范和纪律**：分支策略是什么？提交信息怎么写？合并冲突怎么处理？这些都需要团队一起约定并遵守。

**Git不只是工具，更是协作规范的载体。**

### Code Review：被看见的勇气

第一次被组员Review代码时，心里有点紧张。但看到他们提的反馈，我才意识到：

自己写的代码，和「合格的代码」之间，还有不小的距离。

- 防抖处理缺失 → 重复提交风险
- 变量命名随意 → 可读性差
- 同步API滥用 → 性能隐患

Review让我明白：**代码不仅是写给自己看的，更是写给未来维护它的人看的。**

### 沟通：比代码更重要的事

有一周，接口文档和实际返回不一致，组员花了两小时排查。

这类问题的根源往往不是技术，而是**沟通断层**：

| 问题     | 后果            |
| ------ | ------------- |
| 文档不更新  | 前端按旧文档写，实际对不上 |
| 进度不透明  | 别人以为好了，实际还在开发 |
| 需求变更没说 | 开发了一半发现要改     |

教训：**主动同步、变更通知、具体描述**，比埋头写代码更重要。

## 思维方式的转变

### 学生思维 → 工程师思维

| 维度   | 学生思维   | 工程师思维        |
| ---- | ------ | ------------ |
| 任务完成 | 做完就行   | 做好才行，考虑可维护性  |
| 遇到问题 | 等老师解答  | 主动排查，搜索引擎优先  |
| 代码风格 | 能跑就行   | 考虑可读性、团队协作   |
| 进度管理 | 最后一刻赶工 | 分阶段交付，每日站会同步 |

### 全栈能力的初体验

由于在小组里承担了部分后端接口的开发任务，我有了一次难得的「前端做到后端」的体验。

这让我真正理解：**前后端只是分工不同，本质上都是在处理数据和业务逻辑**。前后端分离的价值，在于让专业的人做专业的事，但彼此需要相互理解。

## 经验沉淀

### 技能层面

| 技能       | 掌握程度        | 后续提升方向            |
| -------- | ----------- | ----------------- |
| uniapp   | 能独立完成中小型项目  | 组件化、性能优化          |
| Git      | 能进行日常协作开发   | 分支策略、CI/CD        |
| 接口设计     | 能编写文档、与后端对接 | RESTful规范、GraphQL |
| 代码Review | 能给别人提有效反馈   | 架构设计经验            |

### 方法层面

- **看文档**：官方文档永远是最好的学习资料
- **写代码**：看十遍不如写一遍，项目驱动学习效率最高
- **排错**：用排除法定位问题，先确认数据源→格式→时序
- **沟通**：主动同步进度，问题描述要具体

### 心态层面

- **保持谦逊**：越是学习，越发现自己不懂的多
- **主动担当**：机会是自己争取来的，不是等来的
- **接受不完美**：交付比完美更重要
- **持续复盘**：每一阶段结束后，问自己「我学到了什么？」

## 结语：这不是终点

2025年1月12日，工匠工坊正式结课。

企业导师在最后一堂课上说的让我印象深刻：

> 「你们现在掌握的，只是冰山一角。真正的企业开发，还有测试、运维、安全、监控……这些都需要在工作中继续学习。」

这让我意识到，这段旅程的意义，不在于掌握了多少技术，而在于：

1. **知道自己能学**——13周入门一个新框架到做出完整项目，这个速度比我想象的要快
2. **知道自己能协作**——从单打独斗到团队配合，这个转变没有想象中那么难
3. **知道自己还要学**——技术之路漫漫，这只是第一站

工匠工坊的故事，到这里就结束了。但这不是终点，而是起点。
