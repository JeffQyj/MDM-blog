## pages.json：uniapp的「宪法」

### 为什么从pages.json开始

在uniapp项目里，**pages.json**是整个应用的「宪法」——它定义了页面路由、窗口样式、导航栏、tabBar等全局配置。企业导师在第一堂课就强调：「先把pages.json搞懂，后面写页面才不会懵。」

在传统前端学习里，我们习惯先写页面再配置路由，而在uniapp中，页面必须在pages.json里先注册才能使用。这种「配置先行」的思路，正是小程序开发的核心理念。

### 基础配置结构

一个最基本的pages.json配置如下：

```json
{
  "pages": [
    {
      "path": "pages/index/index",
      "style": {
        "navigationBarTitleText": "首页"
      }
    }
  ],
  "globalStyle": {
    "navigationBarTextStyle": "black",
    "navigationBarTitleText": "演示",
    "navigationBarBackgroundColor": "#F8F8F8",
    "backgroundColor": "#F8F8F8"
  }
}
```

**配置项解析**：

| 节点            | 作用              | 必填 |
| ------------- | --------------- | -- |
| `pages`       | 页面路径配置数组，第一项为首页 | 必填 |
| `path`        | 页面文件相对路径（不含扩展名） | 必填 |
| `style`       | 页面窗口表现配置        | 可选 |
| `globalStyle` | 全局默认窗口样式        | 可选 |

### 导航栏配置：让你的页面「会说话」

导航栏配置是第一阶段必须掌握的内容。uniapp支持丰富的导航栏自定义能力：

```json
{
  "pages": [
    {
      "path": "pages/detail/detail",
      "style": {
        "navigationBarTitleText": "详情页",
        "navigationBarBackgroundColor": "#ffffff",
        "navigationBarTextStyle": "black",
        "enablePullDownRefresh": true
      }
    }
  ]
}
```

常见的导航栏配置项包括：

| 配置项                            | 说明       | 可选值               |
| ------------------------------ | -------- | ----------------- |
| `navigationBarTitleText`       | 导航栏标题文字  | 字符串               |
| `navigationBarBackgroundColor` | 导航栏背景颜色  | 十六进制颜色值           |
| `navigationBarTextStyle`       | 导航栏文字颜色  | `black` / `white` |
| `enablePullDownRefresh`        | 是否开启下拉刷新 | `true` / `false`  |

:::note
企业导师特别提醒：导航栏高度固定为44px（不包含状态栏），tabBar高度固定为50px（不包含安全区）。这个数值在做布局计算时非常重要。
:::

### tabBar配置：多页面导航的「高速公路」

如果应用有多个主要页面，tabBar是最常用的导航形式：

```json
{
  "tabBar": {
    "color": "#7A7E83",
    "selectedColor": "#3cc51f",
    "borderStyle": "black",
    "backgroundColor": "#ffffff",
    "list": [
      {
        "pagePath": "pages/index/index",
        "text": "首页",
        "iconPath": "static/tabbar/home.png",
        "selectedIconPath": "static/tabbar/home-active.png"
      },
      {
        "pagePath": "pages/user/user",
        "text": "我的",
        "iconPath": "static/tabbar/user.png",
        "selectedIconPath": "static/tabbar/user-active.png"
      }
    ]
  }
}
```

**tabBar的「五不要」**（企业导师反复强调的避坑指南）：

1. **不要**超过5个tab页面
2. **不要**使用本地图片作为icon（图标会被压缩变形）
3. **不要**忘记在pages数组中注册tabBar页面
4. **不要**让iconPath和selectedIconPath指向同一张图
5. **不要**在tabBar页面使用自定义导航栏（除非禁用原生导航栏）

## 模板语法：Vue语法的「uniapp化」

### 数据的「插值」与「绑定」

uniapp的模板语法继承自Vue.js，但做了「小程序化」的调整。企业导师用一个比喻解释这个差异：「Vue像是直接写HTML，而uniapp像是写小程序——有些地方更自由，有些地方要收着点。」

**数据插值**——最基础的动态内容展示：

```vue
<template>
  <view>
    <!-- 静态文本 -->
    <text>欢迎使用</text>

    <!-- 动态数据插值 -->
    <text>{{ title }}</text>

    <!-- 表达式运算 -->
    <text>{{ count + 1 }}</text>

    <!-- 三元运算符 -->
    <text>{{ status ? '已启用' : '已禁用' }}</text>
  </view>
</template>

<script>
export default {
  data() {
    return {
      title: '工匠工坊',
      count: 0,
      status: true
    }
  }
}
</script>
```

### 事件绑定：从「看客」到「参与者」

如果说数据展示是「看」，那事件处理就是让用户真正「参与」进来。uniapp的事件绑定采用「`@`」前缀：

| 事件   | 说明         | 小程序写法            | uniapp写法      |
| ---- | ---------- | ---------------- | ------------- |
| 点击   | 元素被点击时触发   | `bindtap`        | `@tap`        |
| 长按   | 元素被长按时触发   | `bindlongpress`  | `@longpress`  |
| 触摸开始 | 手指触摸屏幕时触发  | `bindtouchstart` | `@touchstart` |
| 触摸结束 | 手指离开屏幕时触发  | `bindtouchend`   | `@touchend`   |
| 输入   | 输入框内容变化时触发 | `bindinput`      | `@input`      |

一个完整的事件处理示例：

```vue
<template>
  <view class="container">
    <!-- 点击事件 -->
    <button @tap="handleClick">点击我</button>

    <!-- 输入事件 -->
    <input
      @input="onInput"
      placeholder="请输入内容"
    />
    <text>输入的内容是：{{ inputValue }}</text>

    <!-- 计数器示例 -->
    <view class="counter">
      <button @tap="decrement">-</button>
      <text>{{ count }}</text>
      <button @tap="increment">+</button>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      inputValue: '',
      count: 0
    }
  },
  methods: {
    handleClick() {
      uni.showToast({
        title: '按钮被点击了！',
        icon: 'none'
      })
    },
    onInput(e) {
      this.inputValue = e.detail.value
    },
    increment() {
      this.count += 1
    },
    decrement() {
      if (this.count > 0) {
        this.count -= 1
      }
    }
  }
}
</script>
```

:::note
企业导师强调的一个关键点：**uniapp中无法直接操作DOM**。所有UI交互必须通过「数据驱动」的方式实现——修改`data`中的数据，由框架负责更新视图。这与传统Web开发的`document.getElementById()`思路完全不同。
:::

## 基础组件：构建界面的「积木」

### view：最通用的容器

`view`组件相当于HTML中的`div`，是最常用的容器组件。它的核心作用是**布局和分组**。

```vue
<template>
  <view class="container">
    <!-- 横向排列 -->
    <view class="row">
      <view class="item">项目1</view>
      <view class="item">项目2</view>
      <view class="item">项目3</view>
    </view>

    <!-- 纵向排列 -->
    <view class="column">
      <view class="card">卡片1</view>
      <view class="card">卡片2</view>
    </view>
  </view>
</template>

<style>
.container {
  padding: 20rpx;
}

.row {
  /* 启用flex布局 */
  display: flex;
  /* 横向排列 */
  flex-direction: row;
  /* 项目之间的间距 */
  gap: 20rpx;
}

.column {
  display: flex;
  flex-direction: column;
  gap: 20rpx;
  margin-top: 20rpx;
}

.item {
  flex: 1;
  text-align: center;
  padding: 20rpx;
  background-color: #f0f0f0;
  border-radius: 8rpx;
}

.card {
  padding: 30rpx;
  background-color: #ffffff;
  border-radius: 12rpx;
  box-shadow: 0 2rpx 8rpx rgba(0, 0, 0, 0.1);
}
</style>
```

### text：文本展示的正确姿势

`text`组件是uniapp中专门用于展示文本的组件，有几个关键特性需要记住：

```vue
<template>
  <view>
    <!-- 基本文本 -->
    <text>这是普通文本</text>

    <!-- 可选择的文本（相当于web的user-select） -->
    <text selectable>这段文字可以被选择复制</text>

    <!-- decode解码（用于显示特殊字符） -->
    <text decode>&lt; &gt; &amp;</text>

    <!-- 结合数据 -->
    <text>你好，{{ username }}</text>
  </view>
</template>
```

:::warning
一个常见的误区：**不要用`view`组件的文本节点来显示文字**。虽然某些平台可能正常显示，但在其他平台可能会出现样式问题。始终使用`text`组件包裹文本内容。
:::

### image：图片的「正确打开方式」

`image`组件用于显示图片，它的默认行为与HTML的`img`标签不同——**默认高度为0，需要手动设置**。

```vue
<template>
  <view>
    <!-- 固定宽高比 -->
    <image
      src="/static/logo.png"
      mode="aspectFit"
      style="width: 200rpx; height: 200rpx;"
    />

    <!-- 宽度自适应，高度自动计算 -->
    <image
      src="/static/banner.jpg"
      mode="widthFix"
      style="width: 100%;"
    />

    <!-- 裁剪模式：保留比例填满容器 -->
    <image
      src="/static/avatar.jpg"
      mode="aspectFill"
      style="width: 120rpx; height: 120rpx; border-radius: 60rpx;"
    />
  </view>
</template>
```

**`mode`属性详解**：

| mode值         | 说明                   | 适用场景  |
| ------------- | -------------------- | ----- |
| `scaleToFill` | 缩放图片到填满容器（会变形）       | 不推荐   |
| `aspectFit`   | 保持宽高比，完整显示图片（常用于商品图） | 最佳    |
| `aspectFill`  | 保持宽高比，裁剪填满容器（常用于头像）  | 裁剪填充  |
| `widthFix`    | 宽度不变，高度自动计算          | 响应式图片 |
| `heightFix`   | 高度不变，宽度自动计算          | 响应式图片 |

### button：不只是「按钮」

`button`组件是uniapp中功能最丰富的组件之一，它不仅仅是按钮，还能直接调用微信/支付宝的开放能力：

```vue
<template>
  <view>
    <!-- 普通按钮 -->
    <button @tap="handleClick">默认按钮</button>

    <!-- 主色调按钮 -->
    <button type="primary" @tap="handlePrimary">主要操作</button>

    <!-- 成功色调 -->
    <button type="success">成功</button>

    <!-- 开放能力：获取用户信息 -->
    <button
      open-type="getUserInfo"
      @getuserinfo="onGetUserInfo"
    >
      微信登录
    </button>

    <!-- 开放能力：分享 -->
    <button open-type="share">分享给好友</button>

    <!-- 开放能力：联系客服 -->
    <button open-type="contact">联系客服</button>

    <!-- 带图标的按钮 -->
    <button @tap="handleScan" class="scan-btn">
      <text class="icon">&#xe600;</text>
      扫码
    </button>
  </view>
</template>

<script>
export default {
  methods: {
    handleClick() {
      console.log('按钮被点击')
    },
    handlePrimary() {
      uni.showToast({ title: '主要操作', icon: 'success' })
    },
    onGetUserInfo(e) {
      if (e.detail.userInfo) {
        console.log('获取到用户信息：', e.detail.userInfo)
      }
    },
    handleScan() {
      uni.scanCode({
        success: (res) => {
          console.log('扫码结果：', res.result)
        }
      })
    }
  }
}
</script>

<style>
.scan-btn {
  display: flex;
  align-items: center;
  justify-content: center;
  gap: 10rpx;
}

.icon {
  font-size: 40rpx;
}
</style>
```

## 样式系统：rpx与Flex布局

### rpx：uniapp的「响应式密码」

uniapp使用`rpx`作为默认尺寸单位，它是**响应式像素**（responsive pixel）的缩写。rpx的核心理念是：**让不同屏幕尺寸自动适配**。

```vue
<template>
  <view class="container">
    <!-- 使用rpx单位，自动适配不同屏幕 -->
    <view class="box-100">100rpx</view>
    <view class="box-200">200rpx</view>
    <view class="box-full">全宽</view>
  </view>
</template>

<style>
.box-100 {
  width: 100rpx;
  height: 100rpx;
  background-color: #1AAD19;
  margin-bottom: 20rpx;
}

.box-200 {
  width: 200rpx;
  height: 200rpx;
  background-color: #367fa9;
  margin-bottom: 20rpx;
}

.box-full {
  width: 750rpx;  /* 750rpx = 屏幕宽度100% */
  height: 200rpx;
  background-color: #f0f0f0;
}
</style>
```

**为什么是750rpx？**

微信小程序设计时规定：**屏幕宽度统一为750rpx**。无论在什么尺寸的手机上，750rpx都等于屏幕的实际物理像素宽度。这意味着：

| 设备屏幕宽度                    | 1rpx = ? 物理像素 |
| ------------------------- | ------------- |
| iPhone 6 (375px)          | 0.5px         |
| iPhone 12 (390px)         | 0.52px        |
| iPhone 15 Pro Max (430px) | 0.573px       |

使用rpx单位，元素会按比例自动缩放，真正实现「一套代码，多端适配」。

### Flex布局： uniapp的「万能布局器」

企业导师在第一阶段就反复强调：「**在uniapp里，优先使用Flex布局**。」这不是建议，而是最佳实践——因为某些平台（如nvue）只支持Flex布局。

Flex布局的两大核心概念：

**1. 主轴与交叉轴**

```vue
<template>
  <view class="demo-container">
    <!-- 默认：主轴水平，交叉轴垂直 -->
    <view class="flex-row">
      <view class="item">A</view>
      <view class="item">B</view>
      <view class="item">C</view>
    </view>

    <!-- 主轴垂直，交叉轴水平 -->
    <view class="flex-column">
      <view class="item">D</view>
      <view class="item">E</view>
      <view class="item">F</view>
    </view>
  </view>
</template>

<style>
.demo-container {
  padding: 20rpx;
}

.flex-row {
  display: flex;
  flex-direction: row;  /* 主轴水平方向 */
}

.flex-column {
  display: flex;
  flex-direction: column;  /* 主轴垂直方向 */
  margin-top: 20rpx;
}

.item {
  width: 100rpx;
  height: 100rpx;
  background-color: #00AAFF;
  color: #fff;
  display: flex;
  align-items: center;
  justify-content: center;
  margin-right: 20rpx;
}
</style>
```

**2. 主轴对齐方式（justify-content）**

| 属性值             | 效果          | 生活类比         |
| --------------- | ----------- | ------------ |
| `flex-start`    | 靠左对齐        | 教室排座靠左       |
| `flex-end`      | 靠右对齐        | 教室排座靠右       |
| `center`        | 居中对齐        | 教室排座居中       |
| `space-between` | 两端对齐，项目之间等距 | 书架两端对齐，书等距摆放 |
| `space-around`  | 项目两侧间距相等    | 桌子的「左右空间」相同  |
| `space-evenly`  | 项目之间和两侧间距相等 | 均匀分布         |

**3. 交叉轴对齐方式（align-items）**

| 属性值          | 效果             |
| ------------ | -------------- |
| `flex-start` | 交叉轴起点对齐        |
| `flex-end`   | 交叉轴终点对齐        |
| `center`     | 交叉轴居中对齐        |
| `stretch`    | 项目被拉伸至填满容器（默认） |
| `baseline`   | 项目基线对齐         |

## HBuilderX：从「小白」到「熟手」

### 为什么是HBuilderX

起航篇提到，项目选择uniapp的原因之一是「**HBuilderX提供完整的IDE支持，开箱即用**」。经过第一阶段的学习，我对这句话有了更深的体会。

HBuilderX是DCloud官方出品的IDE，专门为uniapp开发优化。它的核心优势在于：

| 优势        | 说明                                |
| --------- | --------------------------------- |
| **语法提示**  | pages.json、manifest.json等配置文件自动提示 |
| **组件提示**  | 基础组件标签和属性自动补全                     |
| **rpx单位** | 原生支持rpx单位，无需额外配置                  |
| **真机调试**  | 一键连接手机，实时预览效果                     |
| **多端发行**  | 一套代码，可发布到不同平台                     |

### 项目创建：从零开始

使用HBuilderX创建uniapp项目非常简单：

1. **新建项目**：点击「文件」→「新建」→「项目」
2. **选择模板**：选择「uni-app」→「默认模板」
3. **填写信息**：输入项目名称、保存路径
4. **等待创建**：IDE自动生成项目结构

创建完成后的目录结构：

```
my-project/
├── pages/              # 页面目录
│   └── index/
│       └── index.vue   # 首个页面文件
├── static/             # 静态资源目录
│   └── tabbar/
├── App.vue             # 应用入口文件
├── main.js             # Vue实例入口
├── manifest.json       # 应用配置
├── pages.json          # 页面路由配置
└── uni.scss            # 全局样式变量
```

### 运行与调试：真机体验

HBuilderX的调试体验是它最吸引人的地方。企业导师在演示时，只要点击「运行」→「运行到手机或模拟器」→「运行到手机或模拟器」，就能看到手机上的实时效果。

**三种运行方式**：

| 运行方式         | 适用场景   | 特点             |
| ------------ | ------ | -------------- |
| **Chrome预览** | 快速开发调试 | 即开即用，适合H5页面    |
| **手机预览**     | 真机体验   | 需要数据线连接或WiFi配对 |
| **模拟器预览**    | 无手机时使用 | 模拟真实设备环境       |

**WiFi配对步骤**（企业导师教的小技巧）：

1. 确保手机和电脑在同一WiFi网络下
2. 手机开启「USB调试」并允许调试
3. HBuilderX中运行「WiFi调试」
4. 根据提示在手机上的HBuilderX配对
5. 配对成功后，每次只需保持手机和电脑在同一网络，即可无线调试

:::note
企业导师特别提醒：WiFi配对需要在HBuilderX 4.71及以上版本。旧版本需要使用adb命令配对。另外，如果遇到设备检测不到的问题，可以尝试执行`adb.exe devices`命令检查设备连接状态。
:::

## 实战demo：把知识串起来

### 任务卡片的实现

第一阶段的结课作业是一个「任务卡片列表」，要求展示多条任务数据，包含标题、状态、截止日期等信息。这个demo把前面学到的知识全部串联起来：

```vue
<!-- pages/task/task.vue -->
<template>
  <view class="task-page">
    <!-- 页面标题 -->
    <view class="page-header">
      <text class="title">我的任务</text>
      <text class="count">共 {{ taskList.length }} 项</text>
    </view>

    <!-- 任务列表 -->
    <view class="task-list">
      <view
        v-for="(task, index) in taskList"
        :key="index"
        class="task-card"
        :class="{ completed: task.status === 'done' }"
        @tap="onTaskTap(task)"
      >
        <!-- 左侧：勾选框 -->
        <view class="checkbox" @tap.stop="toggleTask(index)">
          <text v-if="task.status === 'done'" class="check-icon">✓</text>
        </view>

        <!-- 中间：任务信息 -->
        <view class="task-content">
          <text class="task-title">{{ task.title }}</text>
          <text class="task-date">截止：{{ task.deadline }}</text>
        </view>

        <!-- 右侧：优先级标签 -->
        <view
          class="priority-tag"
          :class="'priority-' + task.priority"
        >
          {{ getPriorityText(task.priority) }}
        </view>
      </view>
    </view>

    <!-- 空状态 -->
    <view v-if="taskList.length === 0" class="empty-state">
      <text class="empty-icon">📋</text>
      <text class="empty-text">暂无任务</text>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      taskList: [
        { title: '完成uniapp基础语法笔记', status: 'done', deadline: '2024-10-25', priority: 'high' },
        { title: '整理HBuilderX快捷键列表', status: 'done', deadline: '2024-10-28', priority: 'medium' },
        { title: '编写任务卡片demo', status: 'pending', deadline: '2024-11-05', priority: 'high' },
        { title: '阅读uniapp官方文档', status: 'pending', deadline: '2024-11-10', priority: 'low' },
        { title: '准备阶段考核', status: 'pending', deadline: '2024-11-15', priority: 'high' }
      ]
    }
  },
  methods: {
    toggleTask(index) {
      const task = this.taskList[index]
      task.status = task.status === 'done' ? 'pending' : 'done'
    },
    onTaskTap(task) {
      uni.showModal({
        title: task.title,
        content: `截止日期：${task.deadline}\n优先级：${this.getPriorityText(task.priority)}`,
        showCancel: false
      })
    },
    getPriorityText(priority) {
      const map = { high: '高', medium: '中', low: '低' }
      return map[priority] || '未知'
    }
  }
}
</script>
```

这个demo涉及的核心知识点：

| 知识点              | 对应内容        |

| `v-for`循环        | 渲染任务列表      |
