:::tip
2026年3月，朋友找我帮忙完成他的毕业论文。彼时的我，甚至不知道LaTeX怎么发音。但就是因为答应了朋友，一场跨越困难的学习之旅就此展开。这不是一篇纯粹的LaTeX教程，而是一个关于责任与成长的故事。
:::

## 缘起：一句拜托

朋友拜托我，用LaTeX帮他完成论文的写作以及排版。

说实话，那一刻我脑子里闪过的是：「LaTeX是什么？」

说实话，我完全不会LaTeX，在此之前从未听说过。但朋友之托，确实不好拒绝。再加上对自己的学习能力还有几分自信——不就是个排版工具吗，学一学就会了。

于是我回复：「行，我试试。」

朋友那边是真的急，毕业论文的Deadline就在眼前。而我刚刚答应的是「帮他写完并排版」——这意味着我不仅需要学会用LaTeX，还要帮他把论文内容写出来，任务十分艰巨。

:::note
朋友之托，难以推辞。加上对自己的学习能力有几分底气，想着不就是个工具吗，学一学总会掌握的。但没想到，这个「学一学」，差点把我折腾到崩溃。
:::

## 噩梦开局：安装配置的坑

既然答应了，第一步当然是安装LaTeX。

但我完全低估了这个「安装」的分量。

当时的我对LaTeX一无所知，连「发行版」这个概念都没有。问了一圈、搜了一圈，得到的建议是「安装最全的那个版本」。于是我下载了一个集成了几乎所有宏包的LaTeX发行版，安装包体积高达七八个G。

然后就是漫长的等待。

一小时过去了，进度条缓慢爬行。两小时过去了，还在继续。三小时、四小时……

我开始意识到不对劲，但为时已晚——安装已经卡在某个环节，想取消又怕系统出问题，只能干等着。

最后算下来，四五个小时过去了，安装进度还卡在原地。

:::warning
这里踩的第一个坑：并不是「最全的就是最好的」。LaTeX的发行版有很多种，不同的发行版适合不同的场景。初学者很容易像我一样，以为选一个体积最大的就是最优解。
:::

安装失败，系统还差点被我搞出毛病。

## 到处求救：AI、B站、在线网站

安装失败后，我开始四处求教。

首先想到的是问AI。我打开对话框，输入：「LaTeX怎么安装？」AI给出了一长串解释，提到了TeX Live、MikTeX、CTeX等多个发行版，还贴心地给出了安装建议。

然后我去B站搜教程。确实找到了不少，但看下来一个问题始终困扰着我：为什么每个人说的都不一样？有的推荐TeX Live，有的推荐MikTeX，还有的说直接用在线平台就行。

说到在线平台，我也试过几个。Overleaf、Authorea这类在线LaTeX编辑器确实方便——不用安装，打开浏览器就能用。但实际用起来，又是这报错那报错，有的是编译超时，有的是缺失宏包，折腾了半天也没搞明白。

那几天，我陷入了一个奇怪的循环：

- 安装失败 → 问AI → 得到更多选项 → 更迷茫
- 搜教程 → 发现说法不一 → 不知道该信谁
- 用在线平台 → 报错 → 换平台 → 还是报错

每一件事都在消磨我的信心。

## 转机：B站教程与MikTeX

就在快要放弃的时候，我在B站刷到了一个教程。

那个UP主用很简短的话说了几句建议，其中一句大概是：「初学者别一上来就装TeX Live，太大了。用MikTeX就行，轻量级，按需安装宏包。」

就这一句话，让我找到了方向。

顺着这个思路，我去了解了MikTeX。它的安装包只有几百兆，安装过程简洁明了，一路下一步即可。装完之后，它自带了一个简单的包管理器，当编译时遇到缺失的宏包，会提示你是否要自动下载。

这不就是我一直想要的东西吗？

:::note
后来我才明白，LaTeX的学习曲线之所以陡峭，很大程度上是因为前置知识太多。你需要知道发行版是什么、编译器是什么、宏包又是什么——而这些概念，对于一个从未接触过的人来说，就是第一道门槛。那个B站教程的UP主，可能只是随口一说，但对我来说是真正的救命稻草。
:::

安装好MikTeX的那一刻，我长舒了一口气。终于可以开始干活了。

## LaTeX入门：文档结构与基础排版

安装只是开始，真正的挑战是使用。

朋友的论文模板发过来后，我打开一看——密密麻麻的代码，完全不知道从何下手。那时候的心态很微妙：已经答应了朋友，不能说不会；但眼前这些东西，确实看不懂。

怎么办？一个字：学。

### 文档的基本结构

LaTeX文档分为**导言区**和**正文区**两部分。

导言区用来设置全局参数，比如文档类型、引入宏包等；正文区则是文档的实际内容。

```latex
% 导言区
\documentclass[UTF8]{ctexart}  % 文档类型：中文文章
\usepackage{graphicx}           % 引入宏包：图片支持
\usepackage{hyperref}           % 引入宏包：超链接

% 正文区
\begin{document}
这里是正文内容
\end{document}
```

### 章节结构

LaTeX用命令来创建章节层级：

```latex
\section{第一章}           % 一级标题
\subsection{1.1 节}        % 二级标题
\subsubsection{1.1.1 小节} % 三级标题
```

### 列表环境

列表有两种：无序列表和有序列表。

```latex
% 无序列表
\begin{itemize}
  \item 第一项
  \item 第二项
  \item 第三项
\end{itemize}

% 有序列表
\begin{enumerate}
  \item 第一步
  \item 第二步
  \item 第三步
\end{enumerate}
```

### 字体与对齐

字体操作需要用大括号分组：

```latex
\textbf{加粗文本}     % 黑体
\textit{斜体文本}    % 斜体
\underline{下划线}   % 下划线

% 居中对齐
\begin{center}
  这是居中的内容
\end{center}
```

### 换行与分段

LaTeX中，换行和分段是两回事：

- `\\` 表示换行但不另起段落
- 空行 表示另起一个新段落

```latex
这是第一段的第一行\\
这是第一段的第二行

这是第二段。
```

## 进阶排版：表格、图片、公式

基础掌握了，接下来是真正的挑战——朋友的论文模板里，有大量的表格、图片和公式。

### 表格制作

LaTeX的表格需要用 `tabular` 环境，配合 `table` 环境来添加标题和标签。

```latex
\begin{table}[htbp]  % htbp = here, top, bottom, page
\centering
\caption{实验数据汇总}
\label{tab:experiment}
\begin{tabular}{l|c|r}  % l=左对齐 c=居中 r=右对齐
  \hline
  实验组 & 样本数 & 成功率 \\
  \hline
  A组 & 100 & 85\% \\
  B组 & 100 & 92\% \\
  \hline
\end{tabular}
\end{table}
```

:::note
`[htbp]` 参数用来控制表格的浮动位置：`h` 表示尽量放在当前位置，`t` 表示放在页面顶部，`b` 表示放在底部，`p` 表示单独一页。如果你发现表格总是被放到奇怪的位置，可以试试加上 `!` 来强制执行，比如 `[!htbp]`。
:::

如果需要更专业的三线表效果，可以使用 `booktabs` 宏包：

```latex
\usepackage{booktabs}

\begin{table}[htbp]
\centering
\caption{对比实验结果}
\begin{tabular}{lccc}
  \toprule
  模型 & 准确率 & 召回率 & F1值 \\
  \midrule
  ResNet50 & 92.3\% & 89.1\% & 90.7\% \\
  VGG16 & 88.7\% & 91.2\% & 89.9\% \\
  \bottomrule
\end{tabular}
\end{table}
```

### 图片插入

图片需要引入 `graphicx` 宏包，并使用 `figure` 环境：

```latex
\usepackage{graphicx}

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.8\textwidth]{example.png}
  \caption{系统架构图}
  \label{fig:architecture}
\end{figure}
```

如果需要并排插入多张图片，可以配合 `subfigure` 宏包：

```latex
\usepackage{subfig}

\begin{figure}[htbp]
  \centering
  \subfloat[模型A结果]{\includegraphics[width=0.45\textwidth]{model_a.png}}
  \quad
  \subfloat[模型B结果]{\includegraphics[width=0.45\textwidth]{model_b.png}}
  \caption{不同模型的效果对比}
  \label{fig:comparison}
\end{figure}
```

:::warning
LaTeX对图片格式有一定偏好，矢量图（如 `.eps`、`.svg`）效果最好，位图（如 `.jpg`、`.png`）也可以使用，但清晰度会受限。
:::

### 公式编辑

LaTeX最强大的功能之一就是公式排版，这是Word等工具很难比拟的。

**行内公式**（在正文中的公式）：

```latex
根据欧拉公式 $e^{i\pi} + 1 = 0$，我们可以得到...

% 推荐使用以下写法代替 $$
根据欧拉公式 \( e^{i\pi} + 1 = 0 \)，我们可以得到...
```

**行间公式**（单独占一行的公式）：

```latex
\begin{equation}
  E = mc^2
  \label{eq:einstein}
\end{equation}
```

**公式对齐**（多行公式对齐）：

```latex
\begin{align}
  f(x) &= x^2 + 2x + 1 \\
       &= (x+1)^2
\end{align}
```

**常用公式命令**：

| 命令 | 效果 | 说明 |
|------|------|------|
| `\frac{分子}{分母}` | \(\frac{a}{b}\) | 分式 |
| `x^{上标}` | \(x^{n}\) | 上标 |
| `x_{下标}` | \(x_{i}\) | 下标 |
| `\sum_{i=1}^{n}` | \(\sum_{i=1}^{n}\) | 求和 |
| `\int_{a}^{b}` | \(\int_{a}^{b}\) | 积分 |
| `\alpha \beta \gamma` | \(\alpha \beta \gamma\) | 希腊字母 |
| `\sin \cos \tan` | \(\sin \cos \tan\) | 三角函数 |

**矩阵**：

```latex
\usepackage{mathtools}

\begin{equation*}
A = \begin{bmatrix}
1 & 2 & 3 \\
4 & 5 & 6
\end{bmatrix}
\end{equation*}
```

### 文献引用

学术论文少不了文献引用。LaTeX通常使用BibTeX来管理参考文献。

首先创建一个 `.bib` 文件，比如 `refs.bib`：

```bibtex
@article{einstein1905,
  author = {Einstein, A.},
  title = {Zur Elektrodynamik bewegter Körper},
  journal = {Annalen der Physik},
  year = {1905}
}
```

然后在正文中引用：

```latex
爱因斯坦在1905年提出了狭义相对论 \cite{einstein1905}。

% 在文档末尾添加参考文献
\bibliographystyle{plain}
\bibliography{refs}
```

## 摸着石头过河：一步步完成论文

掌握了这些知识后，我开始逐行研究朋友的论文模板。

朋友的论文模板比标准模板复杂得多，用到了很多自定义命令和宏包。遇到不懂的命令，就去查；遇到报错了，就去搜；搜不到答案，就去问AI。

整个过程像极了编程入门时的状态——不是在写代码，而是在查文档、搜错误、问问题。

表格跨页了？查一下 `longtable` 宏包。图片位置不对？调整一下 `[htbp]` 参数。公式编号乱了？检查一下 `align` 环境的用法。

一个一个问题解决，一个一个坑踩过去。

最终，朋友的毕业论文排版完成了。

## 感悟：责任感的重量

事后回想这段经历，如果问我学到了什么，我会说：

**学到了LaTeX，但不止于LaTeX。**

这一路磕磕绊绊，踩的坑一个接一个。为什么能坚持下来？最核心的原因只有一个——我答应了朋友。

如果只是我自己想学LaTeX，遇到安装失败那天，可能就直接放弃了。费那么大劲装一个软件，结果还报错，算了，不学了。

但因为是对朋友的承诺，所以我不能放弃。安装失败就换发行版，编译报错就逐个解决，查不到答案就换关键词继续搜。

这个过程中，我深刻体会到了一个道理：

:::tip
人的成长一定在于解决问题，而解决问题的前提，是承担责任。
:::

很多时候，我们不是做不到，而是因为没有责任在身上，所以遇到困难就退缩了。但如果一件事「必须」由你来完成，「必须」由你来扛，你会发现自己的潜力远超想象。

我开始理解那些成长极快的人。他们为什么敢于主动把责任往自己身上揽？因为他们知道——责任就是压力，压力就是动力，动力就是成长。

放弃了，朋友也不会怪我。但我自己会知道，我选择了逃避。

所以不能放弃，也不愿意放弃。

---

*受人之托，忠人之事。这是我在这段LaTeX学习之旅中，收获的最宝贵的一课。*
