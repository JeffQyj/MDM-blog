## 复杂度分析：算法的「照妖镜」

学习数据结构与算法，不得不提的一个概念就是「时间复杂度和空间复杂度」。

### 大O表示法：增长的「量级」

| 复杂度 | 名称 | 典型示例 |
|-------|------|---------|
| O(1) | 常数时间 | 数组随机访问 |
| O(log n) | 对数时间 | 二分查找 |
| O(n) | 线性时间 | 顺序查找 |
| O(n log n) | 线性对数时间 | 归并排序、快速排序 |
| O(n²) | 平方时间 | 冒泡排序、选择排序 |
| O(2ⁿ) | 指数时间 | 递归的斐波那契 |
| O(n!) | 阶乘时间 | 排列问题 |

一个直观的比喻：

> 如果你有 10 个苹果：
> - O(1)：一次猜中（你是神仙）
> - O(log n)：每次排除一半（猜数字游戏）
> - O(n)：一个一个数
> - O(n²)：把每个苹果跟其他每个苹果比较一遍

### 复杂度分析的「潜规则」

1. **忽略常数项**：O(2n) 和 O(n) 是一回事
2. **忽略低阶项**：O(n² + n) 简化为 O(n²)
3. **关注最坏情况**：除非特别说明，一般分析最坏情况
4. **空间换时间**：有时增加内存消耗能换来时间上的优化

:::note
记住这些复杂度等级很重要，它将帮助我们评估每种数据结构和算法的「效率」。接下来的学习中，我们会不断用到这些概念。
:::

## 从「数组」开始的旅程

一切要从最基础的数据结构说起。

数组，大概是每个程序员接触的第一个「容器」。它的概念简单得不能再简单：连续内存空间里存放相同类型的元素，通过下标随机访问。

```c
// C语言定义一个整型数组
int arr[5] = {1, 2, 3, 4, 5};

// 访问第三个元素（下标从0开始）
printf("%d", arr[2]);  // 输出 3
```

当时觉得这东西太简单了，简直不值一提。但后来才明白，越是简单的东西，越藏着深奥的秘密。

:::note
数组最大的特点是什么？是「随机访问」的时间复杂度为 O(1)。无论你想访问哪个元素，只要知道下标，就能一步到位。但代价是什么呢？插入和删除操作的时间复杂度是 O(n)——因为你需要移动大量元素来保持连续性。
:::

这就是数据结构的「哲学」：**没有完美的结构，只有适合的场景**。

## 链表：从「指针恐惧」到豁然开朗

如果说数组是「不动如山」，那链表就是「动若流水」。

链表的核心思想是：每个节点不仅存储数据，还要存储指向下一个节点的「指针」（或者在Java里的「引用」）。

```c
// C语言实现单链表节点
struct Node {
    int data;
    struct Node* next;  // 指向下一个节点的指针
};

// 创建节点
struct Node* createNode(int value) {
    struct Node* newNode = (struct Node*)malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = NULL;
    return newNode;
}
```

### 我在链表上踩过的那些坑

学习链表的过程并不顺利，有几个坑至今记忆犹新：

**坑一：空指针的恐惧**

```c
// 错误示例：没有判断空指针
struct Node* p = head->next;
printf("%d", p->data);  // 如果head本身就是NULL，程序直接崩溃
```

**坑二：循环链表导致的「死循环」**

```c
// 错误示例：遍历时没有正确判断结束条件
while (p != NULL) {
    printf("%d", p->data);
    p = p->next;  // 如果链表有环，这里永远不会遇到 NULL
}
```

**坑三：malloc后的内存泄漏**

```c
// 创建节点后忘记 free
struct Node* node = createNode(10);
// ... 使用节点 ...
free(node);  // 必须手动释放内存
```

:::warning
链表的核心价值在于：插入和删除操作只需要 O(1) 的时间复杂度（前提是你已经找到了那个节点）。代价是访问任何一个节点都需要 O(n) 的时间——你必须从头开始一个一个往下找。
:::

### 数组 vs 链表：一张表说清楚

| 对比维度 | 数组 | 链表 |
|---------|------|------|
| 存储方式 | 连续内存空间 | 分散内存，通过指针连接 |
| 随机访问 | O(1)，直接按下标计算地址 | O(n)，必须从头遍历 |
| 插入/删除 | O(n)，需移动元素 | O(1)，只需修改指针 |
| 内存利用率 | 高（无额外开销） | 低（需要存储指针） |
| 是否需要扩容 | 是，超出容量需重新分配 | 否，按需分配 |

一个形象的比喻：

> 数组像排排坐的教室，每个人必须紧挨着坐；链表像散落在操场上的同学，靠「喊一声」找到下一个人。

## 栈与队列：两种「有序限制」

### 栈：后进先出的世界

栈大概是世界上最守规矩的数据结构——它只允许在一端进行操作，这一端叫做「栈顶」。

```c
// 栈的基本操作
typedef struct {
    int data[100];
    int top;  // 栈顶指针，-1 表示空栈
} Stack;

// 入栈
void push(Stack* s, int x) {
    s->data[++s->top] = x;
}

// 出栈
int pop(Stack* s) {
    return s->data[s->top--];
}
```

栈的应用场景多得超乎你的想象：

| 应用场景 | 具体例子 |
|---------|---------|
| 函数调用 | 每调用一个函数，就把它压入调用栈 |
| 表达式求值 | 中缀转后缀、后缀表达式计算 |
| 括号匹配 | 检验表达式中的括号是否合法 |
| 深度优先搜索 | DFS遍历图和树 |
| 撤销操作 | 编辑器的 Ctrl+Z |

### 队列：先进先出的排队系统

队列与栈相反，它遵循「先进先出」（FIFO）的原则——第一个进入的，第一个出来。

```c
// 队列的基本操作（顺序队列）
typedef struct {
    int data[100];
    int front;  // 队首
    int rear;   // 队尾
} Queue;

// 入队
void enqueue(Queue* q, int x) {
    q->data[q->rear++] = x;
}

// 出队
int dequeue(Queue* q) {
    return q->data[q->front++];
}
```

队列的典型应用：

| 应用场景 | 具体例子 |
|---------|---------|
| 任务调度 | 操作系统中的进程排队 |
| 广度优先搜索 | BFS遍历图和树 |
| 消息队列 | 异步通信、任务解耦 |
| 打印队列 | 打印任务按顺序执行 |

### 栈与队列对比

| 特性 | 栈 | 队列 |
|------|-----|------|
| 操作原则 | 后进先出（LIFO） | 先进先出（FIFO） |
| 主要操作 | push、pop | enqueue、dequeue |
| 应用侧重 | 深度优先、递归模拟 | 广度优先、任务调度 |

:::note
为什么理解这两种数据结构很重要？因为它们是很多高级数据结构和算法的「底层支撑」。比如Java的调用栈、计算机的消息队列，它们的思想都源于此。
:::

## 树：层次结构的优雅表达

如果说数组和链表是「一维」的数据结构，那树就是「二维」的了。

### 二叉树：每个节点最多两个「孩子」

二叉树是树结构中最简单、也最重要的形式。每个节点最多有两个子节点：左孩子和右孩子。

```c
// 二叉树节点定义
struct TreeNode {
    int data;
    struct TreeNode* left;   // 左子树
    struct TreeNode* right;  // 右子树
};
```

### 遍历：走进树的「姿势」

这是让我头疼了好一阵子的地方。二叉树的遍历有四种主要方式：

| 遍历方式 | 顺序 | 特点 |
|---------|------|------|
| 前序遍历 | 根 → 左 → 右 | 先访问根节点 |
| 中序遍历 | 左 → 根 → 右 | 二叉搜索树得到有序序列 |
| 后序遍历 | 左 → 右 → 根 | 先处理子节点，再处理根 |
| 层序遍历 | 按层从左到右 | 需要借助队列 |

```c
// 前序遍历（递归版本）
void preOrder(struct TreeNode* root) {
    if (root == NULL) return;
    printf("%d ", root->data);      // 访问根
    preOrder(root->left);           // 遍历左子树
    preOrder(root->right);          // 遍历右子树
}

// 中序遍历（递归版本）
void inOrder(struct TreeNode* root) {
    if (root == NULL) return;
    inOrder(root->left);
    printf("%d ", root->data);      // 访问根
    inOrder(root->right);
}
```

:::warning
递归虽美，但别忘了递归的「代价」——每递归一次，就要消耗一层栈空间。如果树的高度很大，递归深度过深，可能会导致栈溢出（Stack Overflow）。后来我才知道，有一种优化叫「尾递归优化」，但并不是所有编译器都支持。
:::

### 二叉搜索树：树的「排序天赋」

如果二叉树满足「左子树所有节点小于根，右子树所有节点大于根」这个条件，它就变成了二叉搜索树（BST）。

BST 的神奇之处在于：**查找、插入、删除的平均时间复杂度都是 O(log n)**。

```c
// BST插入操作
struct TreeNode* insert(struct TreeNode* root, int value) {
    if (root == NULL) {
        return createNode(value);
    }
    if (value < root->data) {
        root->left = insert(root->left, value);
    } else {
        root->right = insert(root->right, value);
    }
    return root;
}

// BST查找操作
struct TreeNode* search(struct TreeNode* root, int value) {
    if (root == NULL || root->data == value) {
        return root;
    }
    if (value < root->data) {
        return search(root->left, value);
    }
    return search(root->right, value);
}
```

但BST有个「致命缺陷」：如果插入的数据是有序的，树就会退化成链表，查找复杂度退化为 O(n)。这就引出了后续的「平衡二叉树」（AVL树、红黑树）等高级数据结构——那是后话了。

## 图：连接一切的可能性

如果说树是「有根有序」的分层结构，那图就是「无拘无束」的关系网络。

### 图的表示：邻接矩阵 vs 邻接表

```c
// 邻接矩阵表示法
#define MAX_VERTEX 100
int matrix[MAX_VERTEX][MAX_VERTEX];  // matrix[i][j] = 1 表示顶点i到j有边

// 邻接表表示法
struct EdgeNode {
    int vertex;           // 边的终点
    struct EdgeNode* next;
};

struct VertexNode {
    int data;
    struct EdgeNode* firstEdge;
};

struct VertexNode graph[MAX_VERTEX];
```

| 表示方法 | 适用场景 | 空间复杂度 |
|---------|---------|-----------|
| 邻接矩阵 | 频繁判断两点是否相连 | O(V²)，V为顶点数 |
| 邻接表 | 遍历所有邻接边 | O(V+E)，E为边数 |

### 图的遍历：DFS 与 BFS

```c
// 邻接表下的 BFS
void BFS(int start) {
    int visited[MAX_VERTEX] = {0};
    Queue q;
    initQueue(&q);

    visited[start] = 1;
    enqueue(&q, start);

    while (q.front != q.rear) {
        int v = dequeue(&q);
        printf("%d ", v);

        EdgeNode* p = graph[v].firstEdge;
        while (p != NULL) {
            if (!visited[p->vertex]) {
                visited[p->vertex] = 1;
                enqueue(&q, p->vertex);
            }
            p = p->next;
        }
    }
}
```

图的应用无处不在：社交网络的好友关系、导航软件的最短路径、编译器依赖分析——说到底都是「图」的延伸。

## 排序算法：八仙过海，各显神通

排序，大概是算法世界里最「卷」的一个领域。五花八门的排序算法，让人眼花缭乱。

### 那些「简单」的排序

#### 冒泡排序：名字好听，实力「菜」

```c
void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n-1; i++) {
        for (int j = 0; j < n-i-1; j++) {
            if (arr[j] > arr[j+1]) {
                swap(&arr[j], &arr[j+1]);
            }
        }
    }
}
```

时间复杂度 O(n²)，空间复杂度 O(1)。名字虽然「泡泡」的，但实际上是最慢的排序之一。

| 排序算法 | 时间复杂度（平均） | 时间复杂度（最坏） | 空间复杂度 | 稳定性 |
|---------|-----------------|-----------------|-----------|-------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 |

:::note
稳定性是什么？稳定排序是指当两个相等的元素时，排序后它们的相对位置不变。比如 `[2, 3, 2, 1]` 排序后，两个 `2` 的相对位置保持不变（第一个 `2` 仍在第二个 `2` 前面）。
:::

### 那些「真正能打」的排序

#### 快速排序：分治思想

快速排序的核心思想是「分治」：选一个「基准」元素，把数组分成两部分，左边都比基准小，右边都比基准大，然后递归处理。

```c
int partition(int arr[], int low, int high) {
    int pivot = arr[high];  // 选最后一个元素作为基准
    int i = low - 1;

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(&arr[i], &arr[j]);
        }
    }
    swap(&arr[i+1], &arr[high]);
    return i + 1;
}

void quickSort(int arr[], int low, int high) {
    if (low < high) {
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}
```

#### 归并排序：分而治之，再合并

归并排序同样采用分治思想，但它的做法是「先分再合」：

```c
void merge(int arr[], int l, int m, int r) {
    int n1 = m - l + 1;
    int n2 = r - m;

    int L[n1], R[n2];
    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int j = 0; j < n2; j++) R[j] = arr[m + 1 + j];

    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) arr[k++] = L[i++];
        else arr[k++] = R[j++];
    }
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}

void mergeSort(int arr[], int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2;
        mergeSort(arr, l, m);
        mergeSort(arr, m + 1, r);
        merge(arr, l, m, r);
    }
}
```

| 排序算法 | 时间复杂度（平均） | 时间复杂度（最坏） | 空间复杂度 | 稳定性 |
|---------|-----------------|-----------------|-----------|-------|
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 |

### 排序算法的「天花板」

一个有趣的事实是：**基于比较的排序算法，最快也只能达到 O(n log n) 的时间复杂度。**

为什么？因为比较排序的本质是一棵决策树。n 个元素有 n! 种可能的排列，要唯一确定其中一种，需要 log₂(n!) 次比较，而 `log₂(n!) ≈ n log₂n`。

这就是所谓的「比较排序的下界」。

## 查找算法：大海捞针的艺术

排序是让数据「有序」，查找是让数据「可寻」。

### 顺序查找：最笨但最通用

```c
int sequentialSearch(int arr[], int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) return i;
    }
    return -1;  // 未找到
}
```

时间复杂度 O(n)，简单粗暴，没有前提条件。

### 二分查找：有序数组的「加速器」

```c
int binarySearch(int arr[], int n, int target) {
    int left = 0, right = n - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;  // 防止溢出

        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

时间复杂度 O(log n)，但前提是**数组必须有序**。

:::warning
二分查找有个容易出错的地方：`mid = (left + right) / 2` 这种写法在 left 和 right 都很大时可能导致整数溢出。正确的做法是 `mid = left + (right - left) / 2`。
:::

## 越学越无知的感悟

回顾2023年6月的这段学习经历，有一个词不断在脑海中浮现：**「敬畏」**。

对算法的敬畏，来自于一次次「想出来却写不出来」的挫败感。

递归的思想看起来简单，但真正写的时候，指针乱飞、栈溢出、边界条件错误——这些问题接踵而至。

对复杂度分析的敬畏，来自于意识到「我的代码还能更快」的紧迫感。

同样的功能，不同的实现，性能可能相差几十上百倍。写代码不能只追求「能跑」，还要追求「跑得好」。

:::tip
数据结构与算法不是用来「炫技」的，而是用来解决实际问题的。数组、链表、栈、队列是工具，快速排序、二分查找也是工具。知道什么时候用什么工具，才是真正的能力。
:::
