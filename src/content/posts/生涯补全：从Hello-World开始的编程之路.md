:::tip
生涯补全的起点，始于 2022 年那个下午。第一次敲下 Hello World，发现自己落后于人的那一刻，也成为了我选择追赶的开始。
:::

## 一切的开端

如果要给「生涯补全」找一个起点，我想应该是 2022 年 9 月 19 日那个下午。

那时候的我，对编程没有任何概念。只知道计算机能做一些事情，但「如何让计算机听我的话」，完全是一片空白。直到那天，我第一次坐在电脑前，打开了一个叫「Dev-C++」的编辑器。

## 写下的第一行代码

老师让我们输入这样几行代码：

```c
#include <stdio.h>

int main() {
    printf("Hello World!");
    return 0;
}
```

我照着屏幕，一个字母一个字母地敲。分号打错位置，括号漏掉一半，编译器的红色报错铺满了半个屏幕。

但当我修正所有错误，按下运行键的那一刻——

黑色的控制台窗口里，安静地跳出了一句：

```
Hello World!
```

## 那种感觉很奇妙

很难形容当时的心情。屏幕上只有几个简单的英文单词，但它们是我亲手敲进去的，是计算机「听」了我的话之后给出的回应。

就像变魔术一样，但又比魔术更真实。因为我理解了它为什么工作——`printf` 是用来输出的，`main` 是程序入口，`return 0` 表示正常结束。

这不是死记硬背，而是真的「懂了」。

## 发现自己落后了

然而，真正触动我的并不是程序运行成功的那一刻。

当我兴奋地把代码敲完，运行出结果之后，我转头问旁边的同学：「你是怎么知道这些代码要这么写的？你怎么知道每个词是什么意思？」

同学很平静地回答我：「我高中学过。」

那一瞬间，我突然意识到——

**我的起点，是落后的。**

有些人早在高中就已经接触过编程，他们站在更高的起点上，而我只是一个完完全全的新手。我甚至连 `#include` 是什么都不知道，更别说 `stdio.h` 代表什么了。

:::note
落后的滋味并不好受，但真正让我思考的是：接下来该怎么办？
:::

## 落后不可怕，不追赶才可怕

说实话，那一刻心里有一点落差。

看着周围的同学信手拈来，而我连最基本的语法都要一个个对照着敲，这种差距是真实存在的。

但我没有让这种落差变成借口。

我想起了中学时学过的一句话：「落后就要挨打。」在编程这件事上，虽然不至于「挨打」，但我知道如果就此放弃，那差距只会越来越大。

**落后不可怕，不追赶才可怕。**

我没有办法改变自己的起点，但我完全可以改变未来的终点。既然已经输在了起跑线，那就用更快的速度追上去。

## 它改变了我什么

那之后，我开始用另一种眼光看待手机 App、网站、甚至游戏。我会想：这个功能是怎么实现的？背后的逻辑是什么？

更重要的是，我意识到**编程是一种创造能力**。它不是少数人的专利，而是任何人都可以学习的技能。别人能学会，我凭什么不能？

:::warning
如果你也觉得自己是「落后」的那一个，请记住：起点低从来不是借口，眼光放远、脚步不停才是正解。
:::

## 写在最后

如果你是编程新手，正在读这篇文章，我想说：

**不要害怕起点低。**

我第一次写代码时，连分号是什么意思都不知道。但就是那一个个小小的「Hello World」，堆砌出了现在的我。

当你觉得自己落后时，不妨把注意力从「差距」转移到「追赶」上。每解决一个 bug，每理解一个概念，都是在向前迈进。
