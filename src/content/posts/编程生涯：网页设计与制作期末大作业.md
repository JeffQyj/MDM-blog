## 缘起：学号背后的"意外惊喜"

2022 年 11 月 20 日，网页设计与制作课程的老师布置了期末大作业。这次大作业的要求很明确：**纯 HTML + CSS 搭建一个纯静态网站**。

具体做什么网站呢？老师根据学号进行分配。而我，被分配到了**学校政务系统官网的复刻**。

当时拿到这个题目时，内心其实是有些懵的。不是因为题目难，而是因为——虽然课堂上学了 HTML 和 CSS 的基础知识，但那些知识点像是散落的珍珠，串不成一条完整的项链。

:::note
后来才明白，这种"知识点散落"的状态，其实是每个初学者都会经历的阶段。关键在于如何把它们串联起来，形成体系。
:::

## 挫折：搭错类型的"乌龙"

正式开始之前，还发生了一个小插曲。

由于没有仔细阅读作业要求，我一开始搭的竟然是**电商网站复刻**——想着做个炫酷的购物页面，展示各种商品卡片和购物车功能。

结果可想而知，老师打回重做。

那一刻有些沮丧，但更多的是警醒：**读懂需求比开始动手更重要**。这次乌龙让我深刻记住了这个道理，直到现在，每次开始一个新项目，我都会先反复确认需求文档，把每一个细节都弄清楚。

:::warning
教训：动手之前，先把需求理解透彻。磨刀不误砍柴工，切勿急于求成。
:::

## 攻坚：从头复习到渐入佳境

被打回后，我重新审视了政务系统官网的特点：庄重、简洁、层级分明、信息密度高。

由于平时学习不够系统，面对这个陌生的题目，我决定先**从头到尾快速复习一遍 HTML + CSS**。这一次，不是零散地看，而是带着"我要做一个完整项目"的目标去学。

复习的过程中，我逐渐找到了感觉。以下是我当时梳理的 HTML + CSS 学习路径和技术体系：

### HTML：标签体系的从零梳理

学习 HTML，最核心的是理解**标签是信息的结构化表达**。每个标签都有其语义含义，浏览器根据这些含义渲染页面。

| 标签类型 | 核心标签 | 使用场景 |
|---------|---------|---------|
| **文档结构** | `<html>`、`<head>`、`<body>` | 页面骨架，`<head>` 放元信息，`<body>` 放可见内容 |
| **头部信息** | `<meta>`、`<title>`、`<link>`、`<script>` | SEO、页面标题、外部资源引入 |
| **文本内容** | `<h1>`~`<h6>`、`<p>`、`<span>`、`<strong>`、`<em>` | 标题、段落、行内强调 |
| **列表** | `<ul>`、`<ol>`、`<li>` | 无序列表、有序列表 |
| **链接与媒体** | `<a>`、`<img>`、`<video>`、`<audio>` | 跳转、图片、音视频 |
| **表格** | `<table>`、`<thead>`、`<tbody>`、`<tr>`、`<td>` | 表格数据展示 |
| **表单** | `<form>`、`<input>`、`<label>`、`<select>`、`<button>` | 用户输入与提交 |
| **语义化** | `<header>`、`<nav>`、`<main>`、`<section>`、`<footer>`、`<aside>` | 明确区块含义，提升可读性和 SEO |

:::note
当时最容易混淆的是 `<div>` 和 `<span>` 的区别。简单来说：`<div>` 是块级元素（独占一行），`<span>` 是行内元素（不独占一行）。而语义化标签（header、nav 等）则是为了代码可读性，能用语义化标签就别全用 div。
:::

政务系统官网的搭建中，我深刻体会到了语义化的好处：导航栏用 `<nav>`、文章区域用 `<main>`、侧边栏用 `<aside>`——这样即使不看样式，页面结构也一目了然。

### CSS：样式体系的从零梳理

CSS 的学习，则需要层层递进。我的学习路径是这样的：

#### 第一层：选择器与优先级

```css
/* 基础选择器 */
div { }           /* 标签选择器 */
.title { }        /* 类选择器 */
#header { }       /* ID 选择器（唯一性） */
[disabled] { }    /* 属性选择器 */

/* 组合选择器 */
.header .logo { }      /* 后代选择器：空格 */
.nav > li { }          /* 子选择器：> */
h1 + p { }             /* 相邻兄弟选择器：+ */

/* 优先级：内联样式 > ID > 类 > 标签 > 通配符 */
```

当时踩的坑：**选择器优先级混乱**。比如写了 `.container .box { color: red; }` 却不生效，后来才明白是优先级不够高，或者被其他规则覆盖了。解决方法是用开发者工具检查样式来源。

#### 第二层：盒模型与布局

盒模型是 CSS 最核心的概念之一：

```css
/* 标准盒模型：width = 内容 + padding + border + margin */
box-sizing: content-box;

/* IE 盒模型（现代项目常用）：width = 内容区域本身 */
box-sizing: border-box;

margin: 10px;           /* 外边距 */
padding: 10px;          /* 内边距 */
border: 1px solid #ccc;  /* 边框 */
```

布局则是 CSS 的重头戏。2022 年的时候，主流布局方式有：

| 布局方式 | 适用场景 | 核心属性 |
|---------|---------|---------|
| **标准文档流** | 简单堆叠 | 默认块级/行内 |
| **浮动（Float）** | 文字环绕、经典两栏 | `float: left/right` |
| **定位（Position）** | 固定导航、覆盖层 | `position: relative/absolute/fixed` |
| **Flexbox** | 一维布局（导航栏、卡片组） | `display: flex` |
| **Grid** | 二维布局（整个页面框架） | `display: grid` |

:::note
Flexbox 当时花了我不少时间才理解透彻。核心概念就两个：**主轴（main axis）**和**交叉轴（cross axis）**，配合 `justify-content` 和 `align-items` 控制对齐。记住这两个轴，很多布局问题就迎刃而解了。
:::

政务系统官网中，顶部导航栏用 Flexbox 实现左右分布，轮播图区域用 Grid 实现图片网格，表单区域则结合了浮动和定位。

#### 第三层：样式细节与优化

| 样式属性 | 常用值 | 注意事项 |
|---------|-------|---------|
| **颜色** | `#fff`、`rgb(255,255,255)`、`rgba(0,0,0,0.5)` | 透明色用 rgba |
| **背景** | `background-color`、`background-image` | 背景图与颜色叠加 |
| **字体** | `font-size`、`font-weight`、`font-family` | 建议用相对单位 rem |
| **边框圆角** | `border-radius` | `50%` 变圆形 |
| **阴影** | `box-shadow` | 浮层效果常用 |
| **溢出控制** | `overflow: hidden/auto` | 清除浮动、滚动区域 |
| **显示切换** | `display: none/block/flex` | 控制元素显隐 |

当时还学到了一个重要的**清浮动技巧**：

```css
/* 方法一：使用 clear 属性 */
.clearfix::after {
    content: '';
    display: block;
    clear: both;
}

/* 方法二：父元素设置 overflow: hidden */
.container {
    overflow: hidden;
}
```

政务系统官网中，侧边栏和主内容区的布局就需要清浮动来避免父容器塌陷。

#### 第四层：响应式基础

当时还没有系统学习移动端适配，但通过大作业也了解了一些响应式基础：

```css
/* 媒体查询 */
@media screen and (max-width: 768px) {
    /* 大屏幕样式 */
}

/* 视口单位 */
width: 100vw;   /* 占视口宽度 */
height: 100vh;  /* 占视口高度 */
```

### 学习感悟：知识点串联成体系

复习的过程中，我最大的收获是：**HTML 和 CSS 不是孤立的，而是配合工作的**。

- HTML 负责**结构**，CSS 负责**样式**，JavaScript 负责**行为**
- 语义化 HTML 让 CSS 选择器更清晰，SEO 效果更好
- 盒模型是所有布局的基础，Flexbox 和 Grid 只是更高级的布局工具

当这些散落的知识点被一个具体项目串联起来时，那种感觉就像是拼图终于完整了——这就是我获得的**核心收获**。

## 成果：政务系统官网的诞生

经过一段时间的搭建，我的政务系统官网终于成型。

当时由于时间紧张加之水平有限，网站功能比较基础，主要包括：

- 首页轮播图区域
- 导航栏与各级菜单
- 政务公开栏目展示
- 底部信息公示

虽然界面远称不上精致，但整体结构完整，符合政务网站的基本规范。

:::note
现在回看，那个网站可能满是稚嫩和不成熟，但正是从"能看"到"好看"的这个过程，让我真正理解了前端的价值。
:::

## 感悟：体系化学习的重要性

这次大作业给我最大的启发，就是**体系化学习的重要性**。

之前学 HTML 和 CSS，都是跟着教程走一步看一步，学完就忘。而通过这次项目，我第一次体验了"以终为始"的学习方式：

```
目标驱动 → 带着问题学习 → 边学边做 → 查漏补缺 → 形成闭环
```

这种方式让零散的知识点不再是孤岛，而是彼此关联、互相支撑的知识网络。后来的 React、Vue 等框架学习，我都在不自觉中沿用了这种方法。
