## 起航：一切从课堂开始

2022 年 9 月下旬，我第一次在课堂上接触到前端这个词。

那时候老师布置了一个小 demo 作业，要求用 HTML 和 CSS 做一个简单页面。我照着课本敲了几行代码，运行浏览器——居然真的显示出了文字和样式。那一刻的感觉很奇妙，像是打开了一扇新世界的大门：**原来自己也可以构建一个网站，展示给别人看。**

正是这种新奇感驱使我课后立刻去 B 站搜索教程，开始了真正的自学之路。

## 青涩岁月：连 JS 都不知道

回头看那段时光，最直接的感受是：**不知道自己不知道什么**。

当时的我只知道「前端」这个名词，却完全不了解它的全貌。写页面时遇到一个常见需求——并排展示几个内容块，第一反应是去搜索「如何让内容并排」，然后稀里糊涂地用上了 `<table>` 表格标签来实现布局。

现在想来啼笑皆非：**CSS Flexbox 或者 Grid 才是正确的解法，但当时根本不知道这些技术方案的存在。**

更让人汗颜的是，写了那么多页面，我甚至不知道 JavaScript 是什么。在我的认知里，前端就是「写 HTML + CSS」，页面只要能展示内容就够了。

## 熄灯后的兴奋

自学劲头最足的那段时间，发生了件小事。

某天晚上宿舍已经熄灯，我却毫无困意。脑子里还在想着白天写的那个粗糙页面，兴奋得睡不着。于是翻身下床，打开电脑，借着屏幕的光继续鼓捣。

那个页面效果确实很粗糙——配色土气、布局混乱、没有任何交互可言。但每加一个标签、每调一个样式，都让我觉得离「真正的网页」更近了一步。

:::note
那时候的快乐很简单：能跑起来的代码就是好代码，能显示内容的页面就是好页面。
:::

## 后来的后来

后来的故事大家都能猜到。

随着学习的深入，我逐渐知道了：
- **JavaScript** 原来是前端交互的核心
- **CSS Flexbox / Grid** 可以优雅地实现各种布局
- 还有很多**组件库**（如 Element UI、Ant Design）可以快速搭建页面

站在今天看当初的自己，只会觉得：**多么青涩，又多么珍贵。**

## 写在最后

回头看这段经历，有几点感悟：

| 维度 | 当时的做法 | 现在的理解 |
|------|-----------|-----------|
| 学习方式 | 遇到什么搜什么，缺乏体系 | 先搭建知识框架，再按图索骥 |
| 技术视野 | 只知道 HTML/CSS，不知道 JS | 技术栈是整体的，三件套缺一不可 |
| 心态 | 求快求成，恨不得马上写出「正经网站」 | 基础功夫扎实比什么都重要 |

**前端三件套的学习**，与其说是一段技术积累，不如说是一次认知刷新。它让我明白：编程世界里，「知道自己不知道什么」本身就是一种进步。
